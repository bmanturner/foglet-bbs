## Critical Gaps

### 1. RichRow is not yet reusable for visible non-ThreadList state glyphs

Affected requirement: `RICHROW-01`.

Phase 20 verification reports that `RichRow` accepts arbitrary state atoms but only renders visible glyphs for `:unread`, `:sticky`, and `:locked`. Generic future states are preserved as blank fixed-width cells, which does not satisfy the reusable rich-row primitive contract.

Evidence:

- `.planning/phases/20-rich-rows-and-thread-flow/20-VERIFICATION.md`
- `lib/foglet_bbs/tui/widgets/list/rich_row.ex`
- `test/foglet_bbs/tui/widgets/list/rich_row_test.exs`

### 2. Phase 22 has no verification artifact

Affected requirements: `READER-01`, `READER-02`, `READER-03`, `READER-04`.

Phase 22 summaries claim all reader requirements were completed, and the integration checker found PostReader wiring through `PostCard.reader_parts`. The milestone audit still cannot mark those requirements satisfied because `.planning/phases/22-post-reader-facelift/22-VERIFICATION.md` is missing.

Evidence:

- `.planning/phases/22-post-reader-facelift/22-01-SUMMARY.md`
- `.planning/phases/22-post-reader-facelift/22-02-SUMMARY.md`
- `.planning/phases/22-post-reader-facelift/22-03-SUMMARY.md`
- no `.planning/phases/22-post-reader-facelift/22-VERIFICATION.md`

### 3. Phase 25 verification is stale

Affected requirement: `SYSOP-01`.

The Phase 25 verification report still says Sysop SITE, LIMITS, USERS, and SYSTEM smoke blocks are missing. Current code now contains those blocks, so this is likely a stale artifact, not an implementation blocker. It still prevents clean milestone archival until Phase 25 is re-verified or the artifact is regenerated.

Evidence:
