Plan 02 layers the boxed Navigation/Oneliners panels on top of this data layer; Plan 03 layers the size-contract assertions.

Doing the data layer in isolation also keeps tests green between waves.

Output: Refactored `main_menu.ex` exposing `visible_destinations/1` and `visible_actions/1`; extended `main_menu_test.exs` covering role visibility for the destinations list, command-bar non-duplication, and command-bar action visibility (O/H/↑↓).

    def visible_destinations(user) do
      account = if ShellVisibility.account_visible?(user), do: [{"A", "Account"}], else: []
      moderation = if ShellVisibility.moderation_visible?(user), do: [{"M", "Moderation"}], else: []
      sysop = if ShellVisibility.sysop_visible?(user), do: [{"S", "Sysop"}], else: []

      @base_destinations ++ account ++ moderation ++ sysop ++ [@logout_destination]
    end

    ```elixir
    @spec visible_actions(map()) :: [%{label: String.t(), commands: [map()]}]
    def visible_actions(state) do
      user = state.current_user
      oneliners = visible_oneliners(state)

      oneliner_post = if user, do: [@oneliner_action], else: []
      hide_oneliner = if selected_hideable_oneliner(state), do: [@hide_oneliner_action], else: []
      select_oneliner = if oneliners != [], do: [@select_oneliner_action], else: []

      [
        command_group("Actions", hide_oneliner ++ oneliner_post, 10),
        command_group("Select", select_oneliner, 20)
      ]
      |> Enum.reject(&(&1.commands == []))
    end
    ```
