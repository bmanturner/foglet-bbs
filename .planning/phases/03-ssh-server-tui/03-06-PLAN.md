<objective>
Fix three UAT gaps in App routing and SSH event translation:

Gap 4 (minor): Suspended modal Enter key does nothing — app.ex do_update({:key,...})
delegates to the screen module before checking for an active modal. Login's
handle_form_key(%{key: :enter}) always matches when sub-state is :login_form and
returns {:update,...} without checking state.modal, so global_key_handler's modal
dismiss path is never reached.

Gap 5 (major): Board List stuck on "Loading..." — Raxol's Command.task runtime
wraps every task result in {:command_result, inner} before delivering to update/2,
but no {:command_result,...} clause exists in app.ex. All async results (boards,
threads, posts, read_pointers) hit the catch-all and are discarded.

Gap 6 (major): Terminal resize ignored — cli_handler.ex uses Event.window/3 which
creates type: :window, but Raxol's Dispatcher.system_event?/1 only routes type:
:resize to handle_resize_event/2, which sends {:update_size,...} to the Rendering
Engine. :window type goes to the app path, leaving the Rendering Engine dimensions
unchanged.

Purpose: Restore board browsing (major), fix resize (major), and fix modal
dismiss on Enter (minor). Together these close the remaining major blockers from
UAT.

Output: app.ex has modal intercept before screen dispatch and a command_result
re-dispatcher; cli_handler.ex emits :resize events; app_test.exs covers both
new app.ex behaviours.
</objective>

defp do_update({:key, key_event}, state) do
  if state.modal != nil do
    # Modal is active: route key directly to global_key_handler, which
    # contains all modal dismiss / confirm logic. Never delegate to the
    # screen module while a modal is open — screen handlers don't check
    # state.modal and will consume the key silently.
    global_key_handler(key_event, state)
  else
    screen_module = screen_module_for(state.current_screen)

    case screen_module.handle_key(key_event, state) do
      {:update, new_state, commands} ->
        process_screen_commands(new_state, commands)

      :no_match ->
        global_key_handler(key_event, state)
    end
  end
end
```

def handle_ssh_msg({:ssh_cm, _conn, {:window_change, _ch, width, height, _pxw, _pxh}}, state) do
  # Use Event.new(:resize,...) so Raxol's Dispatcher routes this through
  # handle_resize_event/2 (type: :resize is in system_event?/1's allowlist).
  # Event.window/3 produces type: :window, which is NOT a system event and
  # goes to the app update/2 path, leaving the Rendering Engine dimensions
  # unchanged. This is the root cause of Gap 6 (terminal resize ignored).
  event = Raxol.Core.Events.Event.new(:resize, %{width: width, height: height})
  dispatch_events(state.lifecycle_pid, [event])

  if is_pid(state.session_pid) do
    Sessions.Session.set_terminal_size(state.session_pid, {width, height})
  end

  {:ok, %{state | width: width, height: height}}
end
```

From Raxol dispatcher (confirmed via source):
```elixir
# system_event?/1 includes :resize but NOT :window
defp system_event?(%Event{type: type}) do
  type in [:resize, :quit, :focus, :error, :system]
end

# handle_resize_event/2 sends {:update_size, ...} to Rendering Engine
defp handle_resize_event(%{width: width, height: height}, state) do
  if state.rendering_engine do
    GenServer.cast(state.rendering_engine, {:update_size, %{width: width, height: height}})
  end
  send(state.runtime_pid, :render_needed)
  {:ok, %{state | width: width, height: height}, []}
end
```

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|

| T-03-gc2-02 | Denial of Service | SSH window dimensions | accept | Dimensions are integers from SSH protocol; no user-controlled string parsing; extreme values (e.g., 0x0) passed through to Raxol which handles them gracefully |
| T-03-gc2-03 | Elevation of Privilege | modal intercept guard | mitigate | Guard routes all keys to global_key_handler while modal is open; screen handlers cannot consume modal keys silently; modal dismiss requires explicit Enter/Escape/Space |
