<objective>
Extract generic `%Foglet.TUI.Effect{}` interpretation into `Foglet.TUI.App.Effects` while delegating route and modal operations through the routing and modal helpers.
</objective>

<must_haves>
<truths>
- TUI-04: Maintainer can understand and modify `Foglet.TUI.App` through narrow runtime helper modules for routing, modal, subscription, and effect concerns.
- D-01: The effect helper module is `Foglet.TUI.App.Effects`.
- D-02: `Foglet.TUI.App.Effects` is an App-shell runtime helper that may operate on `%Foglet.TUI.App{}` state but must not own durable domain mutations or screen-specific business workflows.
- D-03: `Foglet.TUI.App` remains the Raxol callback integration point and owns when to call effect interpretation.
- D-04: Do not keep App effect helpers solely for old tests.
- D-05: Move effect interpretation coverage to helper-level tests while App-level tests cover integration.
- D-06: Any App effect delegator that remains must have a production/public-boundary reason.
- D-09: Extract generic effects after routing and modal so navigation, modal open/dismiss, modal submit, session, terminal, publish, quit, and task effects can delegate to the correct runtime helpers.
- D-17: Effects owns interpretation of all current `%Foglet.TUI.Effect{}` types while delegating route mutation and modal-submit reducer delivery through routing/modal helpers.
- D-18: Task effects preserve `{:screen_task_result, screen_key, op, {:ok | :error, result}}` wrapper shape and App routes it back as `{:task_result, op, result}`.
- D-19: Domain work remains inside context functions invoked by screen-requested tasks; the effect helper only schedules, dispatches, publishes, notifies sessions, resizes terminal state, or returns runtime commands.
- D-23: Effect contract assertions move to focused helper tests while App integration tests retain modal/effect round trips.
- D-24: Tests assert state/command/message outcomes, not text presence.
</truths>
</must_haves>

<action>
Create `lib/foglet_bbs/tui/app/effects.ex` defining `defmodule Foglet.TUI.App.Effects`.

Implement this public API:

- `apply_effect/2`
- `apply_effects/2`

Move the current `Foglet.TUI.App.apply_effect/2` and `apply_effects/2` behavior into the helper, preserving concrete effect semantics:

- `:navigate` sets `current_screen`, `route_params`, clears `modal`, calls `Routing.init_route_screen_state/3`, then dispatches route entry.
- `:modal` with `{:open, modal}` sets `state.modal`; `:modal` with `:dismiss` clears `state.modal`.
- `:modal_submit` calls `Routing.route_screen_update(state, screen_key, {:modal_submit, kind, payload})` when the target module exports `update/3`; otherwise calls `AppModal.submit_error(state)`.
- `{:session, {:set_user, user}}` must preserve existing session-promotion behavior by re-entering the App shell dispatch path, not by duplicating promotion logic.
- `{:session, {:set_current_user, user}}` sets `current_user`.
- `{:session, {:update_preferences, snapshot}}` merges `timezone`, `time_format`, `theme_id`, and `theme` into `session_context` and calls `Foglet.Sessions.Session.update_preferences/2` when `session_pid` is present.
- `{:session, {:dispatch, message}}` re-enters App shell dispatch.
- Other session payloads send to `session_pid` when present.
- `:terminal` size effect re-enters existing window-change handling.
- `:publish` uses `Phoenix.PubSub.broadcast(FogletBbs.PubSub, topic, message)`.
- `:quit` returns `[Raxol.Core.Runtime.Command.quit()]`.
- `:task` returns one `Foglet.TUI.Command.task/2` command whose function wraps success/rescue/catch into `{:screen_task_result, screen_key, op, {:ok | :error, result_or_reason}}`.
</action>
