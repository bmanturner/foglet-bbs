<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| MainMenu command -> App modal | Target id crosses from advisory UI to command owner. |
| Modal payload -> App | Untrusted reason text enters form submit handling. |
| App -> domain contexts | Current actor and trusted target id are passed to authoritative domain APIs. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-01 | Spoofing | `submit_hide_oneliner` | mitigate | Use `state.current_user`; domain still calls `Bodyguard.permit/4`. |
| T-08-02 | Tampering | Modal payload | mitigate | Accept only `reason`; target comes from `pending_hide_oneliner_id`; no hidden_by/moderator ids from payload. |
| T-08-03 | Repudiation | Hide result handling | mitigate | Successful result must come from `Foglet.Oneliners.hide_entry/3`, which writes durable audit per Plan 08-01. |
| T-08-04 | Information Disclosure | Moderation load | mitigate | Load only through `Foglet.Moderation.workspace_snapshot(current_user)` and render forbidden states for unauthorized actors. |
| T-08-05 | Denial of Service/Layout | reason/body rendering | mitigate | Use `max_length: 240`, visible form errors, existing oneliner limit 5, and layout smoke at 80x24. |
</threat_model>
