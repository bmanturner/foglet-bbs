<objective>
Close BL-03: `Foglet.TUI.Widgets.Modal.Form.init/1` does not validate that
`:fields` is non-empty. Several focus-navigation clauses
(form.ex:195-217, 220-244, 251-273) compute
`n = length(state.fields)` and then `rem(_, n)`. With an empty list this
raises `ArithmeticError` and crashes the entire TUI app process.

Today no caller passes an empty list, but
`Foglet.TUI.Screens.Sysop.SiteForm.State.visible_keys/1` (state.ex:101-108)
already supports filtering, and a future visibility rule could take the
visible set to zero — this is a latent crash waiting for a future
configuration. Replace it with a deterministic, message-bearing
`ArgumentError` at construction time.

Output: `init/1` raises `ArgumentError` with the exact message
"Modal.Form requires at least one field" when called with an empty fields
list. A unit test in `form_test.exs` asserts the raise.
</objective>
