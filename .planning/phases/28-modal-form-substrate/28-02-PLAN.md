<objective>
Add an explicit submit-state machine to `Modal.Form` so accidental double-Enter,
Enter-during-redraw, or held-key bounces cannot invoke a boundary call twice.
While `:submitting`, the form locks all input and shows a visible "Saving…"
status row; the consuming screen drives the transition out via the public
setter `Modal.Form.set_submit_state/2` once async work completes (or fails).

Purpose: FORM-05 — every form-bearing screen downstream (Account Profile,
Account Preferences, Sysop Site) inherits this guarantee, so Phase 29 (Sysop
Site lifecycle, async Config.put error mapping) and Phase 30 (Account async
persistence + flash) can build a real save flow on top of a correct primitive.

Output: Modal.Form's `submit_state` field, the first-clause input lock,
`set_submit_state/2` public setter, the in-flight/saved/error status row,
auto-reset on next event, and complete unit-test coverage.
</objective>

5. Update the Enter clause (formerly Clause 4) to transition `:idle → :submitting` and invoke `on_submit` exactly once (D-05):

```elixir
defp do_handle_event(%__MODULE__{} = state, %{key: :enter}) do
  last_idx = length(state.fields) - 1

  cond do
    state.focus_index == last_idx and state.submit_state == :idle ->
      payload = collect_values(state)
      _ = state.on_submit.(payload)
      {%{state | submit_state: :submitting}, :submitted}

    state.focus_index == last_idx ->
      # submit_state is :saved or {:error, _} — auto-reset already applied above,
      # but the prior call site is the auto-reset preamble. If we got here with
      # state.submit_state == :idle and focus_index == last_idx, the first arm
      # fires. The remaining branch is "Enter on non-last field — advance".
      n = length(state.fields)
      {%{state | focus_index: rem(state.focus_index + 1, n)}, nil}

    true ->
      n = length(state.fields)
      {%{state | focus_index: rem(state.focus_index + 1, n)}, nil}
  end
end
```

(The `cond` collapses cleanly because by the time `do_handle_event/2` runs, `submit_state` is either `:idle` (after auto-reset of `:saved`/`:error`) or `:idle` originally — `:submitting` was short-circuited by the lock clause. The triple-arm cond is intentional for clarity; the runtime collapses to two arms.)

1. Modify `Modal.Form.render/2` (form.ex:188-213) to compute a status row from `state.submit_state` and emit it as the LAST row in the `column []` body. When the status row is non-empty, the footer is suppressed (D-09):

```elixir
def render(%__MODULE__{} = state, opts) do
  %Theme{} = theme = Keyword.fetch!(opts, :theme)

  title_row = text(state.title, fg: theme.title.fg, style: [:bold])
  divider = text(String.duplicate("─", 40), fg: theme.border.fg)

  field_rows =
    state.fields
    |> Enum.with_index()
    |> Enum.flat_map(fn {spec, idx} ->
      focused? = idx == state.focus_index
      field_state = Enum.at(state.field_states, idx)
      render_field(spec, field_state, focused?, state.errors, theme)
    end)

  base_error_rows =
    case Map.get(state.errors, :base) do
      nil -> []
      msg -> [text(msg, fg: theme.error.fg)]
    end

  status_rows = render_status_row(state.submit_state, theme)

  footer_rows =
    cond do
      status_rows != [] -> []
      state.show_footer -> [text("[Enter] Submit   [Esc] Cancel", fg: theme.dim.fg)]
      true -> []
    end

  column [] do
    [title_row, divider] ++ field_rows ++ base_error_rows ++ status_rows ++ footer_rows
  end
end

defp render_status_row(:idle, _theme), do: []

defp render_status_row(:submitting, %Theme{} = theme),
  do: [text("Saving…", fg: theme.dim.fg)]

defp render_status_row(:saved, %Theme{} = theme),
  do: [text("Saved.", fg: status_saved_fg(theme))]

defp render_status_row({:error, msg}, %Theme{} = theme),
  do: [text("Error: #{msg}", fg: theme.error.fg)]

# Use theme.success.fg if the theme exposes a success slot; otherwise fall
# back to theme.accent.fg. The fallback keeps Modal.Form theme-agnostic and
# avoids a hard dependency on Theme schema changes.
defp status_saved_fg(%Theme{} = theme) do
  case theme do
    %Theme{success: %{fg: fg}} when not is_nil(fg) -> fg
    _ -> theme.accent.fg
  end
end
```
