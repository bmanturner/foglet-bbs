<objective>
Close BL-01 (oneliner / hide-oneliner :form modals lock permanently after a
failed submit) and WR-01 (Account ProfileForm/PrefsForm allow-lists drop
:backtab), and document the FORM-05 consumer obligation in
`Foglet.TUI.Widgets.Modal.Form`'s moduledoc so the next reader understands the
lifecycle contract that BL-01 silently violated.

Purpose: Restore the SPEC-mandated guarantee that "Phase 28 must not regress
existing Modal.Form consumers." Today, after a failed oneliner submit the
form's `submit_state` stays `:submitting` forever and the
`form.ex:164-166` lock clause swallows every subsequent key — including
`:escape` — so the only escape from the modal is closing the SSH session.
Resetting `submit_state` to `{:error, msg}` in the two error handlers releases
the lock and the existing auto-reset preamble (`form.ex:181-184`) collapses
the form back to `:idle` on the next non-locked event.

Output: `app.ex` error handlers drive `ModalForm.set_submit_state/2` on every
:form-modal error path; `form.ex` moduledoc states the consumer obligation;
Account ProfileForm/PrefsForm handle_key allow-lists accept `:backtab`;
regression tests cover all three.
</objective>
