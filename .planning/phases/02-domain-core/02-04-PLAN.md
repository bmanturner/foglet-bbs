MDEx API (added as a dep in Plan 01):
- `MDEx.to_html!(markdown_string)` — parses CommonMark + GFM and returns an HTML string

The `Foglet.Markdown` module MUST NOT expose MDEx internals. Single public API:
  `Foglet.Markdown.render(markdown_string) :: String.t()`

ANSI Mapping (D-02):
- bold (`**text**`) → `\e[1mtext\e[0m`
- italic (`*text*`) → `\e[3mtext\e[0m`
- heading (# to ######) → uppercase text with underline `\e[4mTEXT\e[0m` (level 1 = full underline; no distinction between levels beyond uppercase)
- inline code (`` `code` ``) → `\e[2mcode\e[0m`
- code block (fenced or indented) → 2-space indent per line + `\e[2m...\e[0m`
- links (`[text](url)`) → `text (url)`
- images (`![alt](url)`) → alt text only (no URL)
- all other HTML tags stripped from output

Security (T-2-03):
- User-supplied content may include raw ANSI escape sequences (e.g., `\e[31m`)
- Strip raw ESC characters (`\e` / `\x1b`) from the RAW INPUT before passing to MDEx, OR strip them from the MDEx-generated HTML before ANSI transformation
- This ensures users cannot inject their own terminal color codes through post bodies
- The requirement is: ANSI output comes ONLY from parsed AST node types, never from raw user text

CLAUDE.md constraints:
- Never nest multiple modules in the same file
- Predicate functions end with `?`
- This module has no Ecto — it is a pure Elixir transformation module

    ```elixir
    defmodule Foglet.Markdown do
      @moduledoc """
      Renders CommonMark Markdown to ANSI-escaped plain text for terminal display.

      ## ANSI Mapping (D-02)

      | Markdown element   | ANSI output                              |
      |--------------------|------------------------------------------|
      | `**bold**`         | `\e[1mbold\e[0m`                         |
      | `*italic*`         | `\e[3mitalic\e[0m`                       |
      | `# Heading`        | `\e[4mHEADING\e[0m` (uppercased)        |
      | `` `code` ``       | `\e[2mcode\e[0m`                         |
      | ` ```block``` `    | 2-space indent + `\e[2m...\e[0m`        |
      | `[text](url)`      | `text (url)`                             |
      | `![alt](url)`      | alt text only                            |

      ## Security

      Raw ANSI escape sequences in user input are stripped before rendering.
      All styled output originates from the parsed Markdown AST — user text
      cannot inject terminal color codes through post bodies (T-2-03).
      """

      @ansi_reset "\e[0m"
      @ansi_bold "\e[1m"
      @ansi_italic "\e[3m"
      @ansi_dim "\e[2m"
      @ansi_underline "\e[4m"

      @doc """
      Render a Markdown string to ANSI-escaped plain text.

      Accepts CommonMark + GFM Markdown via MDEx. Returns a string suitable for
      terminal output. Raw ANSI escape sequences in the input are stripped.

      ## Examples

          iex> Foglet.Markdown.render("**hello**")
          "\e[1mhello\e[0m"

          iex> Foglet.Markdown.render("# Title")
          "\e[4mTITLE\e[0m\n"

          iex> Foglet.Markdown.render("[click](https://example.com)")
          "click (https://example.com)"

      """
      @spec render(String.t()) :: String.t()
      def render(markdown) when is_binary(markdown) do
        # Strip raw ANSI escape sequences from user input (T-2-03)
        sanitized = strip_ansi(markdown)

        # Parse to HTML via MDEx (CommonMark + GFM)
        html = MDEx.to_html!(sanitized)

        # Transform HTML tags to ANSI sequences
        html
        |> transform_html()
        |> String.trim_trailing()
      end

      # ---------- Private helpers ----------

      # Strip raw ESC characters from user-supplied text before passing to MDEx.
      # This prevents injection of terminal escape sequences through post bodies.
      @spec strip_ansi(String.t()) :: String.t()
      defp strip_ansi(text) do
        # Remove ESC character (\x1b) and anything following it up to and including
        # the final byte of a CSI sequence (letters or ~).
        String.replace(text, ~r/\x1b(\[[0-9;]*[A-Za-z~])?/, "")
      end

      # Walk the HTML output from MDEx and replace known tags with ANSI sequences.
      # Unknown/unsupported tags are stripped (their text content is kept).
      @spec transform_html(String.t()) :: String.t()
      defp transform_html(html) do
        html
        # Headings → uppercase + underline (all levels treated uniformly per D-02)
        |> replace_heading_tags()
        # Bold
        |> String.replace(~r/<strong>(.*?)<\/strong>/s, "#{@ansi_bold}\\1#{@ansi_reset}")
        # Italic
        |> String.replace(~r/<em>(.*?)<\/em>/s, "#{@ansi_italic}\\1#{@ansi_reset}")
        # Inline code
        |> String.replace(~r/<code>(.*?)<\/code>/s, "#{@ansi_dim}\\1#{@ansi_reset}")
        # Code blocks (pre-wrapped code; already handled code tag above for spans;
        # pre > code is the block form in MDEx HTML output)
        |> replace_code_blocks()
        # Images → alt text only (strip src)
        |> String.replace(~r/<img[^>]*alt="([^"]*)"[^>]*>/s, "\\1")
        |> String.replace(~r/<img[^>]*>/s, "")
        # Links → text (url)
        |> String.replace(~r/<a[^>]*href="([^"]*)"[^>]*>(.*?)<\/a>/s, "\\2 (\\1)")
        # Paragraphs → text + newline
        |> String.replace(~r/<p>(.*?)<\/p>/s, "\\1\n")
        # List items
        |> String.replace(~r/<li>(.*?)<\/li>/s, "  • \\1\n")
        # Blockquotes → indented
        |> String.replace(~r/<blockquote>(.*?)<\/blockquote>/s, "  | \\1")
        # Strip remaining HTML tags (keep their text content)
        |> String.replace(~r/<[^>]+>/, "")
        # Decode common HTML entities
        |> decode_html_entities()
      end

      # Replace h1-h6 tags: uppercase the content and wrap with underline ANSI.
      @spec replace_heading_tags(String.t()) :: String.t()
      defp replace_heading_tags(html) do
        Regex.replace(~r/<h[1-6]>(.*?)<\/h[1-6]>/s, html, fn _, content ->
          "#{@ansi_underline}#{String.upcase(content)}#{@ansi_reset}\n"
        end)
      end

      # Replace fenced code blocks (pre > code in MDEx HTML output).
      # Applies dim color and 2-space indent per line.
      @spec replace_code_blocks(String.t()) :: String.t()
      defp replace_code_blocks(html) do
        Regex.replace(~r/<pre><code[^>]*>(.*?)<\/code><\/pre>/s, html, fn _, content ->
          lines =
            content
            |> String.split("\n")
            |> Enum.map_join("\n", fn line -> "  #{@ansi_dim}#{line}#{@ansi_reset}" end)

          lines <> "\n"
        end)
      end

      # Decode common HTML entities produced by MDEx.
      @spec decode_html_entities(String.t()) :: String.t()
      defp decode_html_entities(text) do
        text
        |> String.replace("&amp;", "&")
        |> String.replace("&lt;", "<")
        |> String.replace("&gt;", ">")
        |> String.replace("&quot;", "\"")
        |> String.replace("&#39;", "'")
        |> String.replace("&apos;", "'")
        |> String.replace("&nbsp;", " ")
      end
    end
    ```

    Implementation notes:
    - MDEx renders to HTML (CommonMark spec); we then transform the HTML to ANSI
    - The approach is HTML-in → ANSI-out rather than AST walking, which is simpler and avoids MDEx AST API changes
    - `strip_ansi/1` removes raw `\x1b` escape sequences from input before MDEx sees them (T-2-03)
    - Code blocks: MDEx wraps fenced code in `<pre><code>` which is matched before the inline `<code>` rule
    - `String.replace/3` with regex capture group `\\1` uses Elixir regex replacement syntax (\\1 = first capture)
    - All patterns use `/s` modifier (dotall) so they match across newlines

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User post body → Foglet.Markdown.render/1 | User-supplied markdown is untrusted; raw ANSI escape sequences must be stripped before MDEx processing |
| MDEx NIF → BEAM | MDEx is a Rust NIF; a NIF crash can kill the BEAM scheduler thread. MDEx (comrak) is well-tested and production-ready; risk is low but documented |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-2-03 | Tampering / Info Disclosure | Foglet.Markdown — user ANSI injection | mitigate | `strip_ansi/1` removes `\x1b` and CSI sequences from raw user input before MDEx sees it; all ANSI output is generated exclusively from parsed AST node types (HTML tags) |
| T-2-17 | Denial of Service | MDEx NIF — pathological Markdown input | accept | comrak has protections against deeply nested ASTs; Markdown body length is bounded by `Post` changeset validation (Phase 2 schema). No additional mitigation in this phase. |

## Residual Risks

- `strip_ansi/1` uses a regex that covers standard CSI sequences (`\e[...m`). Exotic terminal escape sequences (OSC, DCS) are not fully covered. The risk is low for BBS use: post bodies are displayed in the BBS TUI which itself controls the terminal — injected sequences would only affect the rendering of that post within the session.
- MDEx NIF crash is a rare but possible event. In Phase 2 this is acceptable; Phase 8 (monitoring) adds crash alerting.
</threat_model>
