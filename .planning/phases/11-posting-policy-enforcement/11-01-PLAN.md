<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| TUI/SSH caller -> `Foglet.Threads.create_thread/3` | Untrusted `user_id`, `board_id`, and attrs enter the domain mutation boundary. |
| `Foglet.Threads` -> `Foglet.Boards.Server` | Crossing this boundary allocates message numbers and persists posting side effects. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-01 | Elevation of Privilege | `Foglet.Threads.create_thread/3` | mitigate | Reload persisted user and board, require active non-deleted poster, enforce `postable_by` matrix, return `{:error, :posting_not_allowed}` before board-server delegation. |
| T-11-02 | Tampering | `Foglet.Boards.Server` allocation path | mitigate | Tests assert rejected thread attempts do not change `boards.next_message_number`, in-memory `next_number`, thread rows, or post rows. |
| T-11-03 | Spoofing | `user_id` argument | mitigate | Do not trust caller-supplied session state; load `Foglet.Accounts.User` by ID inside context and reject missing users. |
| T-11-04 | Information Disclosure | Structured error | accept | `:posting_not_allowed` reveals only policy denial needed for clear terminal feedback; no private board details are added. |
</threat_model>
