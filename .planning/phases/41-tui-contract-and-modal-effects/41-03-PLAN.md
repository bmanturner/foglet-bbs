<objective>
Introduce a first-class modal-submit effect and teach App to route it to target screen reducers as `{:modal_submit, kind, payload}` without process-dictionary transfer.
</objective>

<must_haves>
<truths>
- D-04: `Foglet.TUI.Effect` must expose a typed modal-submit constructor carrying target screen key, submit kind, and payload.
- D-05: Target screen reducers must still receive `{:modal_submit, kind, payload}`.
- D-06: `Modal.Form` submit callbacks must return or emit explicit effect data instead of using process dictionary side channels.
- D-08: Direct App-shell coverage must prove the form-to-App-to-screen round trip.
- D-09: Missing or invalid modal-submit targets must produce a visible failure path.
- QUAL-02: Direct modal-submit round-trip tests are required.
</truths>
</must_haves>

route_screen_update(state, screen_key, {:modal_submit, kind, payload})
