<objective>
Remove the legacy public screen compatibility contract so production screens expose only canonical `init/1`, `update/3`, `render/2`, and optional `subscriptions/2` behavior callbacks.
</objective>

<must_haves>
<truths>
- D-01: `Foglet.TUI.Screen` must not declare `render/1`, `handle_key/2`, or `init_screen_state/1` callbacks or optional callbacks.
- D-02: Production screen modules must not keep public old-callback helpers; reusable setup moves to explicit state constructors, private helpers, or canonical `init/1`.
- TUI-01: Removing the compatibility callbacks must not break production screens.
</truths>
</must_haves>

<action>
Update `lib/foglet_bbs/tui/screen.ex` so its public behavior contract contains only:

- `@callback init(Foglet.TUI.Context.t()) :: local_state()`
- `@callback update(message(), local_state(), Foglet.TUI.Context.t()) :: update_result()`
- `@callback render(local_state(), Foglet.TUI.Context.t()) :: any()`
- `@callback subscriptions(local_state(), Foglet.TUI.Context.t()) :: [String.t()]`
- `@optional_callbacks init: 1, update: 3, render: 2, subscriptions: 2`

Remove the bounded compatibility documentation section, the `@type key_event`, `@type app_state`, `@type command`, `@type handle_key_result`, and the callback declarations for `render/1`, `handle_key/2`, and `init_screen_state/1`.
</action>
<acceptance_criteria>
- `rtk rg -n "@callback (render\\(state|handle_key|init_screen_state)|render: 1|handle_key: 2|init_screen_state: 1|Bounded compatibility" lib/foglet_bbs/tui/screen.ex` exits with no matches.
- `rtk rg -n "@callback (init|update|render|subscriptions)" lib/foglet_bbs/tui/screen.ex` shows only callback declarations for `init/1`, `update/3`, `render/2`, and `subscriptions/2`.
</acceptance_criteria>
</task>
