# Plan 17-05: Widget Visual Contract Remediation

## Problem

Phase 17 completed the theme and mode metadata contract, but it accepted a
weaker definition of "styled" than the v1.3 milestone needs. The phase proved
that touched widgets route colors through `%Foglet.TUI.Theme{}`, but it did not
prove that the widgets satisfy stable visual contracts extracted from the
milestone design.

That is a product gap. This milestone is about making the SSH TUI look like the
screen concepts, and primitive widgets must carry the shape language before
screen conversions can compose them reliably.

Tests must not read, import, parse, or otherwise depend on `SCREENS.md` at

visual decisions as explicit widget contracts in test names, fixtures, and
expected flattened output.

## Gap Audit

| Widget | Current State | Gap Against Milestone Visual Contract | Remediation |
|---|---|---|---|
| `Input.Tabs` | Delegates rendering to Raxol with theme mappings. Tests assert colors and labels. | Does not assert or own the concrete tab strip shape: `▌ Profile   Prefs   SSH Keys   Invites`. Shape can drift behind Raxol defaults. | Render Foglet's tab strip directly or wrap Raxol output behind an exact public contract. Add flattened-text tests for indicator, spacing, active/inactive labels, and absence of unwanted marker text. |
| `Input.RadioGroup` | Renders `> (o)` and `  ( )`. | Uses older scaffolding marks, not the cleaner milestone glyph language. No disabled/error visual state contract. | Add an optional `:variant` or update default to a compact terminal-native marker set. Selected state should not duplicate both `>` and `(o)` unless explicitly required. Add disabled/error tests if supported by API. |
| `Input.Checkbox` | Renders `[x]` and `[ ]`. | Does not match the milestone preference for semantic glyphs such as `✓`, `●`, and `×` on operator surfaces. | Add semantic checked/unchecked/disabled/error marker support while keeping backward compatibility if needed through an `:ascii` option. Tests must prove default marker choice. |
| `Input.Button` | Renders padded text only. | Command/action styling should favor clear labels, key hints, and boring destructive actions. Button output does not distinguish key, label, and destructive treatment structurally. | Split shortcut/key and label into separately styled text runs. Destructive label uses `theme.error`; disabled uses `theme.dim`; no decorative button chrome unless a caller asks. |
| `Input.Menu` | Renders label plus optional shortcut, indented submenus. | Does not support the main-menu/mockup row shape with leading glyph, label column, right shortcut, and disabled/role-gated absence semantics. Contains test-only empty color sentinels. | Remove sentinel text nodes. Support item fields for `:glyph`, `:meta`, and `:shortcut` with width-safe spacing. Keep absent role-gated items caller-owned. |
| `List.SelectionList` | Pure container with optional empty text. | Gives no default visual contract for selected rows; every caller can drift. | Keep pure container, but add an optional default row renderer for simple labels that emits the canonical selected/normal row shape. Existing custom row renderer remains supported. |
| `List.SmartList` | Renders `> label`, search line, and page line. | Missing multi-select marks, compact state glyphs, semantic empty state, and width-safe row shape. Search/pagination affordances are functional but not mockup-like. | Render focused item with a canonical state cluster, render multi-select as `✓`/`◇` or equivalent, use dim metadata for search/page, and add explicit empty/filter-empty states. |
| `Display.Progress` | Renders `[████    ]` plus percentage. | Milestone examples use compact segment bars such as `▰▱▱▱▱`, especially for composer/thread progress. | Add a compact segmented mode and make it the default for low-width widget use. Preserve bracket mode behind an option if existing callers depend on it. Add threshold styles for normal/warning/error/complete. |
| `Progress.Spinner` | Emits only a themed glyph. | No way to render loading text consistently, and no compact/dim secondary treatment. | Add optional message rendering: active glyph in accent/info, message in dim or primary. Keep glyph-only API working. |
| `Modal` | Renders title, divider, message, key hint. | Generic body lacks the stronger heading/action-footer treatment described for modal forms and confirmation/destructive states. | For non-form modals, render a compact title row, message body, and action footer with keys and labels styled separately. Do not take over overlay positioning or `Modal.Form` ownership. |

## Non-Goals

- Do not convert `MainMenu`, `BoardList`, `ThreadList`, `PostReader`,
  composers, Account, Moderation, or Sysop screen bodies.
- Do not implement `List.RichRow`; that remains Phase 20.
- Do not implement `Composer.EditorFrame`; that remains Phase 23.
- Do not implement `Display.Badge`, `Display.KvGrid`, table presets, or
  workspace inspectors; those remain Phase 24.
- Do not redo Chrome V2; that remains Phase 18.

## Tasks

### Task 17-05-01: Add Visual Contract Test Helpers

Files:

- `test/foglet_bbs/tui/widgets/*_test.exs`
- existing `Foglet.TUI.WidgetHelpers`

Work:

- Add or extend helpers for flattened text assertions that preserve enough
  spacing to prove widget shape.
- Add helper assertions for styled runs when a visual contract requires a key,
  glyph, label, or metadata field to use a different theme slot.
- Add a small table of representative milestone glyphs used by these widgets:
  `▌`, `✓`, `×`, `●`, `◇`, `▰`, `▱`.

Acceptance:

- Tests can fail on wrong glyphs or spacing, not just wrong colors.
- Helpers avoid snapshotting entire Raxol maps where targeted flattened-text and
  style-run assertions are clearer.

### Task 17-05-02: Own The Tab Strip Shape

Files:

- `lib/foglet_bbs/tui/widgets/input/tabs.ex`
- `test/foglet_bbs/tui/widgets/input/tabs_test.exs`

Work:

- Make `Tabs.render/2` emit the default canonical tab strip shape directly:

  ```text
  ▌ Profile   Prefs   SSH Keys   Invites
  ```

- Preserve `init/1`, `handle_event/2`, digit shortcut behavior, and
  `{:tab_changed, index}` actions.
- Active indicator uses `Presentation.theme_mappings().tabs.indicator`.
- Active label uses `selected`, bold.
- Inactive labels use `unselected`, no bold, no background.
- Border/container style uses `border` only if the widget owns a container.

Acceptance:

- A test with `["Profile", "Prefs", "SSH Keys", "Invites"]` and active `0`
  asserts flattened text contains exactly one leading `▌ Profile` sequence and
  inactive labels separated by predictable spaces.
- A test with active `2` proves `▌ SSH Keys` moves with active index.
- Tests prove inactive tabs do not receive selected background or bold style.

### Task 17-05-03: Update Selection Controls To Semantic Marks

Files:

- `lib/foglet_bbs/tui/widgets/input/radio_group.ex`
- `lib/foglet_bbs/tui/widgets/input/checkbox.ex`
- `lib/foglet_bbs/tui/widgets/list/selection_list.ex`
- matching tests

Work:

- Replace duplicate scaffolding markers with concise semantic marks where the
  primitive owns the mark.
- Add compatibility options only if existing callers/tests need old ASCII:
  `marker_style: :semantic | :ascii`.
- Radio selected/default output should be compact and not use both `>` and
  `(o)` unless in `:ascii` mode.
- Checkbox semantic mode should use `✓` for checked and a neutral mark for
  unchecked; disabled remains dim.
- `SelectionList` gains an optional simple-label rendering path with selected
  and normal row shape. Custom row renderer remains the default for complex
  screens.

Acceptance:

- Tests assert visible marks and flattened text for selected, unselected, and
  disabled states.
- Existing screen tests still pass without requiring screen body conversion.

### Task 17-05-04: Update Action And Menu Primitives

Files:

- `lib/foglet_bbs/tui/widgets/input/button.ex`
- `lib/foglet_bbs/tui/widgets/input/menu.ex`
- matching tests

Work:

- Button output separates shortcut/key text from action label so command-like
  usage can match the command treatment contract.
- Destructive buttons remain visually boring: clear label, error slot, no
  dramatic decoration.
- Menu supports optional `:glyph`, `:meta`, and `:shortcut` fields in item maps.
- Menu rows align as glyph, label, optional metadata, and shortcut using
  `Foglet.TUI.TextWidth`.
- Remove empty sentinel text nodes used only to expose theme colors.

Acceptance:

- Menu tests assert rows like `● Boards          B` and disabled rows use dim.
- Button tests assert key/shortcut and label are separate styled runs.
- No hardcoded color atoms appear in widget source.

## Success Criteria

- Every widget touched by Phase 17 has at least one visual-shape assertion tied
  to an explicit widget contract encoded in tests.
- `Tabs` default rendering visibly matches the canonical tab strip contract.
- Phase 17 widget tests fail if the implementation regresses to color-only
  styling proof.
- Widgets remain reusable primitives; full screen conversions remain in their
  planned phases.
- All source styling remains theme-routed through `%Foglet.TUI.Theme{}`.
