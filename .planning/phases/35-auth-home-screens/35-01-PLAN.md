|----------|-------------|
| Login reducer -> Accounts/Verification contexts | Screen task closures call domain auth/reset APIs. |
| Task result -> Login reducer | Async auth/reset outcomes re-enter screen-local state. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-35-01 | Elevation of Privilege | Login task effects | mitigate | Domain authentication remains in Accounts/Verification; reducer tests assert success paths emit session/navigation effects only from successful domain results. |
| T-35-02 | Information Disclosure | Reset request honesty | mitigate | Preserve no-email reset behavior without exposing account existence through new reducer branches. |
| T-35-03 | Denial of Service | Task closure failures | mitigate | Use `Effect.task/3` so App wraps success/failure into `{:screen_task_result, :login, op, result}`. |
