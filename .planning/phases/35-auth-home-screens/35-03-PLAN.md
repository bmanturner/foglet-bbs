<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| MainMenu reducer -> Oneliners context | Screen task effects call create/hide/list domain APIs. |
| MainMenu reducer -> Authorization policy | Hide action visibility is advisory UI over Bodyguard-backed domain policy. |
| Modal form -> MainMenu reducer | User-entered body/reason payloads cross from App modal runtime into screen-local logic. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-35-07 | Elevation of Privilege | Hide oneliner action | mitigate | Preserve `Bodyguard.permit?(Authorization, :hide_oneliner, user, :site)` visibility and rely on `Oneliners.hide_entry/3` for authoritative authorization. |
| T-35-08 | Tampering | Pending hide target | mitigate | Store pending hide target in MainMenu.State and validate it exists before `submit_hide_oneliner` task creation. |
| T-35-09 | Denial of Service | Modal submit lock | mitigate | Preserve current form error behavior so failed create/hide submissions release submit state and remain cancelable. |
</threat_model>
