<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Register/Verify reducers -> Accounts/Verification contexts | Screen task effects call durable auth/onboarding APIs. |
| Pending approval result -> session runtime | Register requests termination after presenting pending approval. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-35-04 | Elevation of Privilege | Registration reducer | mitigate | Registration and session promotion results must come only from Accounts/Verification task results. |
| T-35-05 | Denial of Service | Verification resend | mitigate | Preserve cooldown state in Verify.State and tests. |
| T-35-06 | Information Disclosure | Verification failures | mitigate | Preserve existing invalid/expired/no-user user-facing result handling without adding account enumeration branches. |
</threat_model>
