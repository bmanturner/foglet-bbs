<objective>
Close the Phase 43-only verification gap by moving Sysop SITE form initialization out of `Sysop.Render`, so the render module only consumes already-loaded Sysop screen state and never lazy-initializes `SiteForm` or indirectly reads runtime config during render.
</objective>

<gap_source>
Verification report: `.planning/phases/43-large-screen-decomposition/43-VERIFICATION.md`

Failed truth:
- Render modules remain pure over loaded state/context and avoid Repo, PubSub, task starts, durable writes, runtime config reads, and screen-state mutations.

Concrete failure:
- `lib/foglet_bbs/tui/screens/sysop/render.ex` calls `SiteForm.render(SiteForm.init([]), theme)` when `ss.site_form` is nil.
- `SiteForm.init/1` delegates to `SiteForm.State.new/1`.
- `SiteForm.State.new/1` calls `load_drafts/0`, which calls `Config.get!/1`.
- Therefore the Sysop render path can perform runtime config reads.
</gap_source>

Do not modify PostReader bounded-window tests or AppTest reader-window expectations.

</scope_guard>

</truths>
