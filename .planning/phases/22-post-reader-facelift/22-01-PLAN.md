<objective>
Extend `Foglet.TUI.Widgets.Post.PostCard` into the shared reader post unit for Phase 22. Add a small reader-oriented public helper, recommended as `reader_parts/5`, that accepts a post, pre-parsed markdown tuples, width, theme, and `index`/`total` options, then returns separate `header`, `progress`, and `body_lines` pieces. The header and progress are non-scrolling surfaces; only `body_lines` are passed to `Viewport`.

Purpose: Phase 22 must make the reader feel BBS-native without moving visual assembly into `PostReader`. This plan implements the reusable post-unit surface required by D-01 through D-13 and locks it with widget tests before screen integration.

Implements requirements: READER-01, READER-02, READER-03, READER-04.
</objective>

  <action>
    Implement a public helper in `lib/foglet_bbs/tui/widgets/post/post_card.ex`, recommended signature:

    ```elixir
    @spec reader_parts(post_like(), [MarkdownBody.tuple_entry()], pos_integer(), Theme.t(), keyword()) :: %{
            header: any(),
            progress: any(),
            body_lines: [any()]
          }
    def reader_parts(post, tuples, width, %Theme{} = theme, opts \\ [])
    ```

    Required behavior:
    - Read `index = Keyword.get(opts, :index, 0)` and `total = Keyword.get(opts, :total, 1)`. Treat `index` as zero-based.
    - Header string is assembled from exactly these required atoms when present: `Post #{index + 1} of #{total}`, `##{message_number}`, `@#{handle}`, and `"#{age} ago"`. Use `get_handle/1` and `get_time_ago/1`; use `#?`, `@unknown`, or a similarly explicit graceful fallback only for missing fields. The fully-populated normal path must render `Post 1 of 1`, `#42`, `@mina`, and an age token ending with `ago`.
    - Use a single-cell separator between header atoms. Prefer ` • ` per SCREENS.md; another single-cell separator is acceptable only if tests prove compatibility.
    - Render the header with theme-routed slots (`theme.dim.fg`, `theme.title.fg`, `theme.badge.fg`, or `theme.accent.fg`). Do not add hardcoded terminal color atoms.
    - Render progress as compact text `Posts #{index + 1}/#{total}` outside body lines. If adding optional segment glyphs, use `Foglet.TUI.TextWidth.display_width/1` to include them only when `width` can contain compact text plus spacing plus segments. Compact text must always remain.
    - Add gutter support to the body-line path. Recommended implementation: extend `render_body_lines/4` to honor `gutter: true`, compute a body width budget with `TextWidth.display_width(gutter)`, call `MarkdownBody.render_tuples_as_lines/4` with the reduced body width, then wrap each returned row element in `row style: %{gap: 1}` with `[text("│", fg: theme.border.fg), original_row]`. Preserve the existing default behavior when `gutter: false` or omitted.
    - `reader_parts/5` calls `render_body_lines(tuples, width, theme, gutter: true)` and returns only body rows in `:body_lines`; header and progress are separate values.
    - Keep `render/4` and `render_from_tuples/5` behavior backward compatible except for any helper extraction needed internally. Do not put progress into full-card `render/4` unless existing tests are updated intentionally.
  </action>
