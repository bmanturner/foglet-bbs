    1. **Sticky theme slot:** `theme.info.fg` (NOT `theme.badge.fg`). Per
       RESEARCH Open Question 2: `info.fg` is a straight foreground without a
       `bg` override; `badge.fg` has a `bg` key that would draw a colored block
       around the `●` glyph and risk visual noise on themes where the badge
       background contrasts poorly with the row background.

    2. **Read state rendering:** whitespace, NOT `◇`. Per RESEARCH Open
       Question 3: simplifies the render path, avoids one more theme-slot
       decision, and reduces visual noise on read rows. The cluster reserves a
       1-cell space at the read position so columns stay aligned.

    3. **Cluster format and `@cluster_width`:** The cluster is a 4-character
       string of the form `"USL "` where U/S/L are glyph cells (or single spaces
       when absent) and the trailing space is a fixed gap separating cluster
       from title. `@cluster_width = 4`. The four positions hold:
       - position 0: unread glyph or space
       - position 1: sticky glyph or space
       - position 2: locked glyph or space
       - position 3: trailing separator space

       Atoms missing from `state_cluster` render as a 1-cell space at their slot
       so total cluster width never drifts (Pitfall 3). The width invariant is
       guaranteed **by construction** (each cell is either a known 1-cell glyph
       or a single ASCII space). Per 20-REVIEWS.md user decision #1, there is
       NO runtime `^cluster_width = TextWidth.display_width(cluster)` guard
       and NO `fixed_cluster!/1` raising helper. Production code stays soft.
       Plan 20-01 enforces the invariant via property tests.

    4. **Locked glyph:** `⚿` (U+26BF) per D-05 recommendation. `⚑` is forbidden
       (reserved by Phase 19 Moderation MainMenu — Pitfall 2). `🔒`/`🔐` forbidden
       (2-cell — Pitfall 1).

    5. **Render shape:** Returns a `row style: %{gap: 0}` element with a list of
       text children: focus marker, three cluster glyph nodes (one per glyph
       slot), trailing separator space, title+padding, and metadata. Mirrors
       `ListRow.render_with_metadata/6` so `ScreenFrame.render/4` and
       `SelectionList.render/3` compose it the same way.

    6. **Generic state atoms (D-02):** Atoms not in the Phase 20 set
       (`:subscribed`, `:required`, `:category`, etc.) render as visual
       whitespace at the cluster's fixed width. Phase 21/25 may add their own
       glyph mappings without changing the public API.

    7. **Selection-vs-state-glyph precedence (20-REVIEWS HIGH #2 / user
       decision #2):** When `selected: true`, the row uses `theme.selected.bg`
       and `:bold` on every text node so the row visibly highlights. However,
       glyph foregrounds STILL route through their state slot:
       - selected unread glyph: `fg: theme.accent.fg, bg: theme.selected.bg, style: [:bold]`
       - selected sticky glyph: `fg: theme.info.fg, bg: theme.selected.bg, style: [:bold]`

```elixir
defmodule Foglet.TUI.Widgets.List.RichRow do
  @moduledoc """
  Stateless rich-row renderer for Foglet BBS selection lists.

  Renders a single row composed of:

    1. A focus marker (`▌ ` when selected, two spaces otherwise).
    2. A fixed-width state-glyph cluster (unread / sticky / locked).
    3. A left-aligned title (truncated with `…` when needed).
    4. An optional right-aligned metadata trailer.

  All styling routes through `Foglet.TUI.Theme` slots; no hardcoded color
  atoms appear in this module. The cluster reserves a fixed display width
  regardless of which state atoms are present so columns stay aligned across
  all rows.

  ## Public API

      RichRow.render(
        title: "Thread title",
        metadata: "@alice · 3 posts · 2h ago",
        state_cluster: [:unread, :sticky],
        selected: true,
        theme: theme,
        width: 80,           # optional, default 80
        focus_marker: "▌ ",  # optional, default "▌ "
        emphasis: :bold       # optional, e.g. :bold for unread
      )

  ## Required keyword keys

    * `:title`         — String, the row's primary text.
    * `:state_cluster` — list of state atoms (e.g. `[]`, `[:unread]`,
                         `[:sticky, :locked]`).
    * `:selected`      — boolean, true if this row is the currently focused row.
    * `:theme`         — `%Foglet.TUI.Theme{}` for slot lookups.

  ## Optional keyword keys

    * `:metadata`      — String (default `""`). Right-aligned trailer (e.g.
                         `"@handle · N posts · age"`). When `nil` or `""`,
                         no metadata text node is emitted.
    * `:width`         — non-neg integer, total content width in display cells
                         (default 80, matching `ListRow.render_with_metadata/6`).
    * `:focus_marker`  — String, leading marker when `selected: true`
                         (default `"▌ "`).
    * `:emphasis`      — atom, applied to the title text when not selected
                         (e.g. `:bold` for unread).

  ## Supported state atoms

    * `:unread`  — `◆` rendered through `theme.accent.fg` + `:bold`.
    * `:sticky`  — `●` rendered through `theme.info.fg`.
    * `:locked`  — `⚿` rendered through `theme.warning.fg`.

  Atoms outside this set render as 1-cell whitespace, preserving the cluster
  width for future reuse without changing the public API.

  ## Selection vs. state-glyph styling

  When `selected: true`, every text node in the row receives
  `theme.selected.bg` and `:bold` so the row visibly highlights. Glyph
  **foregrounds** still route through their state slot (`accent`, `info`,
  `warning`, `dim`) so that — for example — a selected sticky row's `●`
  remains the sticky color over the selected background. This preserves both
  selection clarity and glyph semantics simultaneously.

  ## Size-contract priority

  At any terminal width:

    1. The leading state cluster always renders fully.
    2. The metadata always renders fully (when supplied).
    3. The title truncates first with `"…"` when needed.
  """

  import Raxol.Core.Renderer.View

  alias Foglet.TUI.TextWidth
  alias Foglet.TUI.Theme

  # Cluster format: "USL " — 3 glyph slots + 1 trailing separator space.
  # Each glyph (or its replacement space) is exactly 1 display cell.
  @cluster_width 4

  @focus_marker "▌ "
  @no_focus_marker "  "
  @ellipsis "…"
  @min_title_length 20

  @glyph_unread "◆"
  @glyph_sticky "●"
  @glyph_locked "⚿"
  @glyph_space " "

  @doc """
  Renders a rich row from a keyword list. See moduledoc for the contract.
  """
  @spec render(keyword()) :: any()
  def render(opts) when is_list(opts) do
    title = Keyword.fetch!(opts, :title)
    state_cluster = Keyword.fetch!(opts, :state_cluster)
    selected = Keyword.fetch!(opts, :selected)
    %Theme{} = theme = Keyword.fetch!(opts, :theme)

    metadata = normalize_metadata(Keyword.get(opts, :metadata, ""))
    width = Keyword.get(opts, :width, 80)
    focus_marker = Keyword.get(opts, :focus_marker, @focus_marker)
    emphasis = Keyword.get(opts, :emphasis, nil)

    marker = if selected, do: focus_marker, else: @no_focus_marker

    cluster_glyphs = build_cluster_glyphs(state_cluster, theme)
    cluster_str = cluster_glyphs |> Enum.map(& &1.char) |> Enum.join()

    {title_body, padding_part} =
      compute_parts(marker, cluster_str, title, metadata, width)

    {title_style, metadata_style, padding_style} =
      styles_for(selected, emphasis, theme)

    cluster_elements =
      Enum.map(cluster_glyphs, fn %{char: char, fg_slot: slot} ->
        text(char, glyph_style(slot, selected, theme))
      end)

    children =
      [
        text(marker, title_style),
        cluster_elements,
        text(title_body, title_style),
        text(padding_part, padding_style)
      ]
      |> List.flatten()

    children =
      if metadata == "" do
        children
      else
        children ++ [text(metadata, metadata_style)]
      end

    row style: %{gap: 0} do
      children
    end
  end

  # ---------------------------------------------------------------
  # Private — cluster construction
  # ---------------------------------------------------------------

  # Returns a list of 4 glyph cells in fixed order:
  #   0: unread slot (◆ or space)  fg_slot: :accent | :dim
  #   1: sticky slot (● or space)  fg_slot: :info   | :dim
  #   2: locked slot (⚿ or space)  fg_slot: :warning| :dim
  #   3: trailing separator space  fg_slot: :dim
  #
  # Width is fixed by construction — every cell is either a known 1-cell glyph
  # (◆ ● ⚿) or a single ASCII space. Plan 20-01 property tests assert the
  # cluster string measures @cluster_width display cells across every state
  # combination (including unknown atoms like :subscribed).
  defp build_cluster_glyphs(state_atoms, _theme) when is_list(state_atoms) do
    [
      cell_for(:unread in state_atoms, @glyph_unread, :accent),
      cell_for(:sticky in state_atoms, @glyph_sticky, :info),
      cell_for(:locked in state_atoms, @glyph_locked, :warning),
      %{char: @glyph_space, fg_slot: :dim}
    ]
  end

  defp cell_for(true, glyph, fg_slot), do: %{char: glyph, fg_slot: fg_slot}
  defp cell_for(false, _glyph, _fg_slot), do: %{char: @glyph_space, fg_slot: :dim}

  # Style for one glyph cell. The :fg always comes from the state slot; the
  # :bg is added only when the row is selected so glyph foreground semantics
  # survive selection (per 20-REVIEWS HIGH #2 / user decision #2).
  defp glyph_style(:accent, true, theme),
    do: [fg: theme.accent.fg, bg: theme.selected.bg, style: [:bold]]

  defp glyph_style(:accent, false, theme),
    do: [fg: theme.accent.fg, style: [:bold]]

  defp glyph_style(:info, true, theme),
    do: [fg: theme.info.fg, bg: theme.selected.bg, style: [:bold]]

  defp glyph_style(:info, false, theme),
    do: [fg: theme.info.fg]

  defp glyph_style(:warning, true, theme),
    do: [fg: theme.warning.fg, bg: theme.selected.bg, style: [:bold]]

  defp glyph_style(:warning, false, theme),
    do: [fg: theme.warning.fg]

  defp glyph_style(:dim, true, theme),
    do: [fg: theme.dim.fg, bg: theme.selected.bg, style: [:bold]]

  defp glyph_style(:dim, false, theme),
    do: [fg: theme.dim.fg]

  # ---------------------------------------------------------------
  # Private — width math
  # ---------------------------------------------------------------

  @spec compute_parts(String.t(), String.t(), String.t(), String.t(), pos_integer()) ::
          {String.t(), String.t()}
  defp compute_parts(marker, cluster_str, title, metadata, width) do
    marker_width = TextWidth.display_width(marker)
    cluster_width = TextWidth.display_width(cluster_str)
    metadata_width = TextWidth.display_width(metadata)
    min_gap = if metadata_width == 0, do: 0, else: 2

    max_title_body =
      max(width - marker_width - cluster_width - min_gap - metadata_width, 0)

    title_body = truncate_title(title, max_title_body)
    title_body_width = TextWidth.display_width(title_body)

    left_width = marker_width + cluster_width + title_body_width

    padding_width =
      (width - left_width - metadata_width) |> max(0) |> min(width)

    padding_part = TextWidth.pad_trailing("", padding_width)

    {title_body, padding_part}
  end

  @spec truncate_title(String.t(), non_neg_integer()) :: String.t()
  defp truncate_title(title, max_width) do
    title_width = TextWidth.display_width(title)

    cond do
      title_width <= max_width ->
        title

      max_width >= @min_title_length ->
        TextWidth.truncate(title, max_width)

      max_width >= 2 ->
        TextWidth.truncate(title, min(max_width, TextWidth.display_width(@ellipsis) + 1))

      true ->
        TextWidth.truncate(title, max_width)
    end
  end

  # ---------------------------------------------------------------
  # Private — style dispatch (title / metadata / padding)
  # ---------------------------------------------------------------

  # Selected: title, metadata, and padding all carry selection styling.
  # Glyph foregrounds are handled separately in glyph_style/3.
  defp styles_for(true, _emphasis, theme) do
    selected_style = Map.get(theme.selected, :style, [:bold])

    sel_kw = [
      fg: theme.selected.fg,
      bg: theme.selected.bg,
      style: selected_style
    ]

    {sel_kw, sel_kw, sel_kw}
  end

  # Unselected + emphasis :bold (unread, not selected): title in accent.fg + :bold.
  defp styles_for(false, :bold, theme) do
    {
      [fg: theme.accent.fg, style: [:bold]],
      [fg: theme.dim.fg],
      [fg: theme.dim.fg]
    }
  end

  # Unselected + no emphasis: plain unselected styling.
  defp styles_for(false, _emphasis, theme) do
    {
      [fg: theme.unselected.fg],
      [fg: theme.dim.fg],
      [fg: theme.dim.fg]
    }
  end

  # ---------------------------------------------------------------
  # Private — metadata normalization
  # ---------------------------------------------------------------

  defp normalize_metadata(nil), do: ""
  defp normalize_metadata(""), do: ""
  defp normalize_metadata(str) when is_binary(str), do: str
end
```

- The cluster fixed-width invariant is guaranteed **by construction** —
  every cell is either a known 1-cell glyph or a single ASCII space.
