**Step 2.** If any stage fails, fix the underlying issue in the file that
generated the warning. Common patterns:

- **Compile warnings (`--warnings-as-errors`):**
  - "module attribute @cluster_width_local was set but never used" → remove
    the redundant alias if my Plan 04 left a leftover. Just keep `@cluster_width 4`.
  - "unused alias" in thread_list.ex → confirm `ListRow` is removed from the alias block.
  - "function thread_state_cluster/2 is unused" → confirm `render_thread_row/4`
    actually calls it.

- **Format diffs:** `rtk mix format` once and re-run precommit.

- **Credo warnings:**
  - "Functions should not be longer than X lines" → extract one of the new
    helpers in RichRow into a smaller private function.
  - "Avoid `Map.get/3` with a default; consider pattern matching" → keep
    `Map.get/3` per RESEARCH Pitfall 5 (struct vs map compatibility) and add
    a `# credo:disable-for-next-line` comment with rationale if Credo insists.
  - "Module attribute @glyph_space appears too short" → ignore unless Credo
    enforces; otherwise rename.

- **Sobelow findings:** Phase 20 changes nothing security-relevant per
  RESEARCH.md Security Domain table. If Sobelow reports a new finding, it's
  almost certainly a false positive from `inspect/2` in test code. Add a
  `# sobelow_skip ["Misc.BinToTerm"]` comment with rationale or use an
  alternative pattern. Never silence with `--ignore-files`.

- **Dialyzer warnings:**
  - Spec mismatch on `RichRow.render/1` — tighten `@spec` to match the
    actual return type (a `Raxol.Core.Renderer.View` element map). Use
    `@spec render(keyword()) :: map()` if `any()` is rejected.
  - "Function will never be called" — confirm the function is reachable
    through `render/1`; if not, remove it.
  - "Pattern match will never succeed" — fix the pattern; do not suppress.

**Step 3.** After every fix, re-run the failing stage to confirm green, then
re-run the full `rtk mix precommit` to confirm no regression in earlier stages.

**Step 4.** Once `rtk mix precommit` exits 0 cleanly, capture the final output
(last 30 lines) and quote it in the plan SUMMARY for evidence.
