<objective>
Implement `subscriptions/2` on the three stateful screens that need PubSub topics: PostReader (thread topic), ThreadList (board topic), BoardList (boards aggregate). All three implementations are additive — existing `update/3` and `render/2` clauses are untouched. Stateless screens are explicitly NOT modified, satisfying D-08's contract that the optional callback works as designed.

Purpose: D-05, D-08, D-09, D-22. Required before App's `build_pubsub_topics/1` is rewritten in Plan 39-05; that plan deletes the App-side topic decoders and unions screen-declared topics instead. Without these implementations, App's new `function_exported?(module, :subscriptions, 2)` check would return false for all screens and the topic set would shrink — breaking SPEC R7 acceptance.

Output: three screen modules each gain a `subscriptions/2` function with two clauses (local-state-first, then route_params-fallback). Three test files have their `@tag :phase39_target` removed from the corresponding `function_exported?/3` pin (added in Plan 39-01) plus three new behaviour pin tests.
</objective>
