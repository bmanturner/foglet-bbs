<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Keyboard/modal input -> App | Untrusted oneliner body enters app state from focused form. |
| App -> Foglet.Oneliners | Authenticated actor and attrs cross into domain validation/persistence. |
| Domain result -> modal UI | Validation/domain errors are transformed into user-visible messages. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-08 | Spoofing | `{:submit_oneliner, payload}` handling | mitigate | Use `state.current_user` as the actor; never accept user_id from modal payload. |
| T-07-09 | Tampering | Modal form payload | mitigate | Domain context remains authoritative for blank/length/same-user validation; TextInput max length is UX only. |
| T-07-10 | Information Disclosure | App modal/rendering | mitigate | Add no hide UI or hidden-entry display; refresh uses `list_recent_visible/1`. |
| T-07-11 | Denial of Service | App load/refresh | mitigate | Use `@oneliner_limit 5` for load command and no live chat subscriptions in Phase 7. |
</threat_model>
