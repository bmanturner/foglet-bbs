<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| SSH key event -> PostComposer screen | Untrusted terminal input is routed by screen-owned key clauses before body input fallback. |
| PostComposer screen -> `Foglet.Posts` context | Reply submission crosses into domain mutation and must keep existing context path. |
| Draft markdown -> preview render | Draft text is rendered for preview only and must not mutate data. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-23-01 | Tampering | Body counter | mitigate | Counter display uses `String.length(draft)` but max-length submit enforcement remains in existing `submit/1` and `enforce_max_len/2`. |
| T-23-02 | Elevation of Privilege | Preview/editor mode | mitigate | Preview and edit both render through EditorFrame; only Ctrl+S reaches existing `submit_reply/4`. |
| T-23-03 | Tampering | Ctrl+S/Ctrl+C and body input | mitigate | Do not reorder `handle_key/2`; existing clause-order tests remain. |
| T-23-04 | Denial of Service | Reply quote at narrow size | mitigate | Quote context is truncated/capped before the editor body and counter can disappear. |
</threat_model>
