<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| PostReader reducer -> Posts context | Load task lists posts for the selected thread. |
| PostReader reducer -> Boards/Threads contexts | Flush task advances read pointers for the visible board/thread. |
| Route params -> read-pointer identity | Board/thread identity from route params becomes the source for flush context. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-37-01 | Tampering | Read-pointer flush | mitigate | Build flush context from route/local state, keep domain writes in Boards/Threads contexts, and clear pending state only on success. |
| T-37-02 | Information Disclosure | Post load route params | mitigate | PostReader only requests posts for the routed thread; domain context remains authoritative for filtering/authorization. |
| T-37-03 | Denial of Service | Render cache warming | mitigate | Warm only selected post cache/viewport on load and navigation, preserving existing width-keyed cache behavior. |
</threat_model>
