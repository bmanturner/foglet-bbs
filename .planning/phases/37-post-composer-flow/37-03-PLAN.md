<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| PostComposer reducer -> Posts context | Submit task calls reply creation with user/thread/board identity. |
| Route params -> reply identity | Thread and board identity come from route/local state, not App current_thread. |
| Task result -> PostComposer reducer | Async domain result updates local state or navigates. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-37-07 | Elevation of Privilege | Reply submission | mitigate | Keep authorization in `Foglet.Posts.create_reply/4`; PostComposer only requests the context mutation. |
| T-37-08 | Tampering | Reply attrs | mitigate | Build attrs from local input and explicit reply_to id only; tests assert exact attrs. |
| T-37-09 | Denial of Service | Submit path | mitigate | Domain call runs through `Effect.task/3`, not synchronously inside key handling. |
</threat_model>
