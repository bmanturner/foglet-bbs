<objective>
Add a context-owned, tombstone-capable bounded reader-window query so PostReader can request small windows without changing historical `list_posts/1` semantics.
</objective>

<must_haves>
<truths>
- POST-01: Very large threads must be readable without PostReader requiring every post in the thread to be loaded eagerly.
- POST-04: Reader/history queries may include tombstones, while list/summary paths that should hide deleted content remain protected elsewhere.
- D-01: The bounded query belongs in `Foglet.Posts`, not in PostReader local slicing.
- D-02: Keep `Foglet.Posts.list_posts/1` tombstone-capable for existing reader/history behavior.
- D-03: `%PostReader.State{}.posts` remains the active window; this plan creates the domain contract consumed later.
- D-13: Soft-deleted posts remain available to reader/history queries.
- D-15: Prefer context-owned query helpers over ad hoc screen predicates.
</truths>
</must_haves>

<threat_model>
- T-44-01 (medium): A bounded query that filters `deleted_at` would break historical reader/tombstone visibility and message-number continuity. Mitigation: tests must soft-delete a post and assert `list_reader_window/2` still returns its row.
- T-44-02 (medium): A cursor query that orders by `inserted_at` or offset can produce unstable windows. Mitigation: order and cursor on `message_number` scoped by `thread_id`.
- T-44-03 (low): Returning raw lists only can hide navigation metadata regressions. Mitigation: return a documented map or `%Foglet.Posts.ReaderWindow{}` with `has_previous?`, `has_next?`, and cursor fields.
</threat_model>

<action>
Add a bounded reader-window result type and public Posts context API.

Create `lib/foglet_bbs/posts/reader_window.ex`:

```elixir
defmodule Foglet.Posts.ReaderWindow do
  @enforce_keys [:posts, :has_previous?, :has_next?]
  defstruct posts: [],
            first_message_number: nil,
            last_message_number: nil,
            has_previous?: false,
            has_next?: false,
            direction: :initial
end
```

Update `lib/foglet_bbs/posts.ex` to alias `Foglet.Posts.ReaderWindow` and expose:

```elixir
@spec list_reader_window(String.t(), keyword()) :: ReaderWindow.t()
def list_reader_window(thread_id, opts \\ [])
```

The function must:

- accept `:limit` with default `50`;
- cap or normalize non-positive limits to a positive bounded value such as `50`;
- accept `direction: :initial | :next | :previous | :last | :around`;
- accept `after_message_number: integer`, `before_message_number: integer`, and `around_message_number: integer` cursors as needed by the direction;
- query only rows where `p.thread_id == ^thread_id`;
- order reader output ascending by `message_number`;
- preload `:user`;
- intentionally not call `QueryHelpers.not_deleted/1`;
- fetch `limit + 1` rows or an equivalent sentinel query so it can set `has_previous?` and `has_next?`;
- return `%ReaderWindow{posts: posts, first_message_number: first_number, last_message_number: last_number, has_previous?: boolean, has_next?: boolean, direction: direction}`.

Keep `list_posts/1` unchanged and document that both `list_posts/1` and `list_reader_window/2` are tombstone-capable reader/history APIs.
</action>
