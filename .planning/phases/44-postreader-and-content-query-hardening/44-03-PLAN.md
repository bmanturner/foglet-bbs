<objective>
Evict stale terminal-width render-cache entries in reducer/state plumbing and strengthen the automated PostReader render-purity guard for the active render module.
</objective>

<must_haves>
<truths>
- POST-02: Resizing during PostReader sessions must not leave stale-width render-cache entries alive for the life of the screen.
- POST-03: Maintainers need automated protection for PostReader render-path purity.
- D-09: Cache eviction belongs in reducer/state plumbing when warming at the current width.
- D-10: Keep cache keys as `{post_id, terminal_width}` unless a simpler equivalent is proved.
- D-11: Preserve or strengthen the automated source/static guard against state writes inside render helpers.
- D-12: Phase 43 moved rendering into `Foglet.TUI.Screens.PostReader.Render`, so the guard must cover that active render boundary.
</truths>
</must_haves>

<threat_model>
- T-44-07 (medium): Eviction implemented in render would hide stale keys while violating render purity. Mitigation: only `warm_cache/4`, `warm_selected_post/2`, or reducer-called helpers may prune `render_cache`.
- T-44-08 (medium): Static guard may keep scanning `post_reader.ex` after render helpers moved to `post_reader/render.ex`. Mitigation: update tests to scan the active render file.
- T-44-09 (low): Cache pruning could delete current-width entries for other posts and create repeated parsing. Mitigation: retain all entries where key width equals current terminal width.
</threat_model>

<action>
Add reducer-side render-cache width eviction.

In `lib/foglet_bbs/tui/screens/post_reader.ex`, add a private helper:

```elixir
defp render_cache_for_width(render_cache, width) do
  render_cache
  |> Enum.reject(fn {{_post_id, cached_width}, _tuples} -> cached_width != width end)
  |> Map.new()
end
```

Call this from `warm_cache/4` before checking or inserting `{post.id, width}`:

```elixir
cache = render_cache_for_width(ss.render_cache, w)
```

If `{post.id, w}` is already in the pruned cache, return `%{ss | render_cache: cache}` so stale widths are removed even on cache hits. If missing, parse and insert into the pruned cache.

Do not add any state writes to `lib/foglet_bbs/tui/screens/post_reader/render.ex`.
</action>
