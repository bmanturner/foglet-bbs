<objective>
Switch PostReader route-entry, jump-last, adjacent-window navigation, and thread-activity reloads to the bounded Posts reader-window contract while keeping `%PostReader.State{}.posts` as the active window.
</objective>

<must_haves>
<truths>
- POST-01: PostReader must read a 1000-post thread without storing all 1000 posts in local state.
- D-03: `%PostReader.State{}.posts` remains the active bounded window.
- D-04: Acceptance must prove fake domain window calls and bounded state shape.
- D-05: `n`, `space`, `page_down`, `p`, `page_up`, `j`, and `k` keep their current semantics.
- D-06: Crossing an active-window boundary requests the adjacent bounded window and lands on the correct boundary post.
- D-07: `load_intent: :jump_last` requests the newest bounded window and selects the newest post.
- D-08: Matching `:thread_activity` reloads use the bounded contract and preserve current reader position as practical.
</truths>
</must_haves>

<threat_model>
- T-44-04 (high): The screen may appear bounded while still calling `list_posts/1` in a task. Mitigation: fake-domain tests must raise on `list_posts/1` and assert `list_reader_window/2` calls.
- T-44-05 (medium): Boundary navigation can regress read-pointer seeding by selecting the wrong post after loading an adjacent window. Mitigation: reducer tests assert selected post id and pending read metadata after next/previous boundary crossings.
- T-44-06 (medium): Thread activity can reset the reader to the first post. Mitigation: preserve selected post by id/message number when the reloaded window still contains it, otherwise choose the nearest available row.
</threat_model>
