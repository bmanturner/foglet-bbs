<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Accounts -> Mailer | User email leaves Foglet only through configured Swoosh delivery. |
| Login/Register -> User | Status and delivery copy influences user expectations and access. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-16 | Repudiation | notification result metadata | mitigate | Return `:attempted`, `:skipped_no_email`, `:not_applicable`, or `{:failed, reason}` from Accounts transitions. |
| T-10-17 | Tampering | delivery failure control flow | mitigate | Persist status before delivery; tests assert failed delivery does not roll back. |
| T-10-18 | Information Disclosure | rejection email/login copy | mitigate | Use generic rejection copy and no internal moderation/history details. |
| T-10-19 | Spoofing | terminal copy | mitigate | Remove unconditional claims that email notification was sent or will be sent. |
| T-10-20 | Elevation of Privilege | rejected login | mitigate | Add `:rejected` login branch that never emits `{:promote_session, user}`. |
</threat_model>
