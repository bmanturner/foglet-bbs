## Precommit Pipeline

Run `mix precommit` before every commit. It chains these checks in order:

```bash
mix compile --warnings-as-errors
mix deps.unlock --unused
mix format
mix credo --strict
mix sobelow --exit Low
mix dialyzer
```

Compile errors, unused deps, formatting drift, any Credo violation, any Sobelow Low+ finding, and Dialyzer type errors all fail the commit. Do not skip any step.

## Module Naming

**Application namespace split:**
- `Foglet.*` — domain/business logic modules (contexts, schemas, OTP workers, TUI)
- `FogletBbs.*` — Phoenix infrastructure (endpoint, router, telemetry, repo, web)

**Pattern:** PascalCase for all module names. One module per file — nesting multiple modules in a single file causes cyclic compile errors.

Examples:
- Context: `Foglet.Accounts`, `Foglet.Boards`, `Foglet.Posts`, `Foglet.Threads`
- Schema: `Foglet.Accounts.User`, `Foglet.Boards.Board`, `Foglet.Posts.Post`
- OTP: `Foglet.Boards.Server`, `Foglet.Sessions.Session`
- TUI screen: `Foglet.TUI.Screens.Login`, `Foglet.TUI.Screens.PostReader`
- TUI widget: `Foglet.TUI.Widgets.Input.TextInput`, `Foglet.TUI.Widgets.List.SmartList`
- Error: `Foglet.Config.InvalidValueError`, `Foglet.Config.UnknownKeyError`

---

## File Naming

- **snake_case** for all file names: `board_list.ex`, `cli_handler.ex`, `ssh_key.ex`
- File name matches the last segment of the module name: `Foglet.Boards.Server` → `server.ex`
- Test files mirror the `lib/` path under `test/` with `_test.exs` suffix:
  - `lib/foglet_bbs/accounts.ex` → `test/foglet_bbs/accounts/accounts_test.exs`
  - `lib/foglet_bbs/tui/screens/login.ex` → `test/foglet_bbs/tui/screens/login_test.exs`
- Sub-state structs live in a subdirectory named after their screen:
  - `lib/foglet_bbs/tui/screens/post_reader/state.ex`
  - `lib/foglet_bbs/tui/screens/new_thread/state.ex`

## Function Naming

- **snake_case** for all function names (enforced by `Credo.Check.Readability.FunctionNames`)
- Predicate functions (return boolean) **must** end in `?`:
  - `require_email_verification?/0`, `color_atom_leaked?/2` (enforced by `Credo.Check.Readability.PredicateFunctionNames`)
- Bang variants (`!`) raise on failure: `get_user!/1`, `Repo.get!/2`, `Config.get!/1`
- Non-bang variants return `nil` or `{:error, reason}`: `get_user/1`, `Config.get/2`
- Context functions follow Phoenix naming conventions: `register_user/1`, `list_ssh_keys/1`, `get_user_by_email/1`, `delete_user/1`

---

## Variable Naming

- **snake_case** (enforced by `Credo.Check.Readability.VariableNames`)
- Unused bindings prefixed with `_`: `_user`, `{_raw, _token}`
- `attrs` is the conventional name for incoming attribute maps to changesets
- Changeset variables use `cs` or `changeset` consistently

## Type Annotations

- Every public function in context modules carries a `@spec`
- Schemas define `@type t :: %__MODULE__{}` for use in specs
- Dialyzer runs with `:error_handling`, `:underspecs`, `:unmatched_returns`, and `:unknown` flags — specs must be accurate
- Private functions do not require `@spec` (Credo.Check.Readability.Specs is disabled)

Example from `lib/foglet_bbs/accounts.ex`:
```elixir
@spec register_user(map()) :: {:ok, User.t()} | {:error, Ecto.Changeset.t()}
def register_user(attrs) do
```

## Documentation

- Every module must have `@moduledoc` (enforced by `Credo.Check.Readability.ModuleDoc`)
- `@moduledoc` explains the module's public API, its consumers, and any design notes (references to planning docs like D-01, BOARD-06, etc.)
- Every public function in context modules and schemas has `@doc`
- Private functions do not require `@doc`
- Section dividers use `# ---------- Section Name ----------` to group related functions within a large module

## Imports and Aliases

- `import Ecto.Query, warn: false` at the top of every module that builds queries
- `Ecto.Changeset` is imported via `use Foglet.Schema` (the shared schema base)
- Aliases are grouped and sorted alphabetically (enforced by `Credo.Check.Readability.AliasOrder`)
- Multi-module aliases use brace syntax: `alias Foglet.Accounts.{SSHKey, User, UserToken}`
- `require Logger` is called inline at the call site, not at module top level — Logger is only required where it's used

## Ecto Schemas

All Ecto schemas use the shared base `Foglet.Schema` via `use Foglet.Schema` (`lib/foglet_bbs/schema.ex`):

```elixir
defmodule Foglet.Accounts.User do
  use Foglet.Schema  # sets UUID primary keys, UUID FKs, utc_datetime_usec timestamps
```

This configures:
- UUID v7 primary key: `@primary_key {:id, Ecto.UUID, autogenerate: true}`
- UUID foreign keys: `@foreign_key_type Ecto.UUID`
- Microsecond UTC timestamps: `@timestamps_opts [type: :utc_datetime_usec]`

**Changeset rules:**
- Separate changesets for each mutation pathway: `registration_changeset/2`, `password_changeset/2`, `role_changeset/2`, `profile_changeset/2`, `deletion_changeset/1`
- Programmatically-set fields (e.g., `user_id`) are set on the struct before changeset construction — never passed through `cast/3`
- Validations extracted into private `validate_*/1` functions: `validate_handle/1`, `validate_email/1`, `validate_password/1`

## Error Handling Patterns

**Context functions return tagged tuples:**
```elixir
{:ok, result}           # success
{:error, %Ecto.Changeset{}} # validation failure
{:error, :not_found}    # explicit not-found atom
{:error, :invalid_credentials}  # domain error atoms
{:error, :expired}      # domain error atoms
```

**`with` chains for multi-step operations inside `Repo.transact/1`:**
```elixir
Repo.transact(fn ->
  with {:ok, updated} <- user |> User.password_changeset(attrs) |> Repo.update() do
    Repo.delete_all(UserToken.by_user_and_contexts_query(user, ["reset_password"]))
    {:ok, updated}
  end
end)
```

**`cond` for mutually exclusive conditions (preferred over nested `if`):**
```elixir
cond do
  user && is_nil(user.deleted_at) && Argon2.verify_pass(password, user.password_hash) ->
    {:ok, user}
  user ->
    {:error, :invalid_credentials}
  true ->
    Argon2.no_user_verify()
    {:error, :invalid_credentials}
end
```

**Bang functions for operations where failure is a programming error:**
- `Repo.get!/2`, `Repo.insert!/1`, `Config.put!/3` — raise on failure
- Used when the caller guarantees preconditions or when a miss is unrecoverable

## Transactions

Multi-step DB operations use `Repo.transact/1` (a thin wrapper over `Repo.transaction/1`):
```elixir
Repo.transact(fn ->
  with {:ok, _edit} <- Repo.insert(edit_cs) do
    post |> Post.edit_changeset(attrs) |> Repo.update()
  end
end)
```

Complex atomic operations involving multiple tables use `Ecto.Multi` (e.g., in `lib/foglet_bbs/boards/server.ex`).

## OTP Patterns

- `GenServer` modules use `@impl true` before each callback implementation
- `start_link/1` takes a keyword list opts and calls `GenServer.start_link(__MODULE__, ..., name: via_tuple(id))`
- Registry-based naming: processes register via `{:via, Registry, {ModuleName.Registry, id}}`
- `DynamicSupervisor` and `Registry` child specs include `name:` option
- OTP child specs for supervised processes use `start_supervised!/1` in tests (never raw `start_link`)

---

## Logging

- `require Logger` called inline, not at module level
- Log levels: `Logger.error/1` for supervisor failures and unexpected states, `Logger.warning/1` for unexpected-but-recoverable events, `Logger.info/1` for dev-time debug paths (e.g., emailing verify codes to console)
- Structured interpolation in log strings: `"[ModuleName] message: #{inspect(value)}"`

## Elixir Gotchas Enforced in Code Review

These are not caught by `mix precommit` — they must be applied manually:

1. **Block expression binding** — bind `if`/`case`/`cond` results to a variable; never reassign inside a block
2. **One module per file** — no nested `defmodule` inside another file's module
3. **Struct field access** — use `struct.field` or `Ecto.Changeset.get_field/2`; never `struct[:field]`
4. **OTP child specs need `name:`** — `DynamicSupervisor` and `Registry` require explicit name
5. **Concurrent enumeration** — use `Task.async_stream/3` with `timeout: :infinity`

## Formatting Rules

Formatter config at `/.formatter.exs`:
- Imports deps: `:ecto`, `:ecto_sql`, `:phoenix`
- Targets: `*.{ex,exs}`, `{config,lib,test}/**/*.{ex,exs}`, `priv/*/seeds.exs`
- Max line length: 120 characters (enforced by `Credo.Check.Readability.MaxLineLength`)
- Formatting is enforced by `mix format` in precommit — no manual style decisions needed
