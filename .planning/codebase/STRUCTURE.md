## Directory Purposes

**`lib/foglet_bbs/`:**
- The core application namespace. Module names match directory paths: `lib/foglet_bbs/boards/server.ex` в†’ `Foglet.Boards.Server`.
- Context modules (`accounts.ex`, `boards.ex`, `threads.ex`, `posts.ex`, `config.ex`) sit at the top of their directory and expose the public API. Schema modules live one level deeper in a subdirectory with the same name.

**`lib/foglet_bbs/tui/`:**
- Everything TUI. Screens are in `screens/`, reusable Raxol view components in `widgets/`. The `app.ex` conductor and supporting abstractions (`screen.ex`, `theme.ex`, etc.) sit at the `tui/` level.
- Screens with non-trivial state have a `state.ex` sibling in a same-named subdirectory (e.g. `post_reader/state.ex`).

**`lib/foglet_bbs/tui/widgets/`:**
- Organized by widget category: `chrome/` (structural frame/bar chrome), `display/` (read-only data display), `input/` (interactive form elements), `list/` (list and selection primitives), `post/` (post-specific renderers), `progress/` (indicators). `compose.ex` and `modal.ex` sit at the `widgets/` level as they are cross-category.

**`lib/foglet_bbs/ssh/`:**
- All Erlang `:ssh` integration: the channel handler (`cli_handler.ex`), daemon supervision (`daemon_owner.ex`, `supervisor.ex`), host-key callback (`key_cb.ex`), and the ETS-based pubkey stash (`pubkey_stash.ex`).

**`lib/foglet_bbs_web/`:**
- Thin Phoenix web layer. Contains only the Endpoint, Router (LiveDashboard + empty `/api`), Telemetry, Gettext, and a JSON error handler. No LiveView modules, no user-facing controllers.

**`lib/mix/tasks/`:**
- Mix tasks for sysop bootstrap and break-glass operations. Named with the `foglet.*` prefix following Mix conventions. Compiled into the `lib` tree (not `test/`), available in all environments.

**`priv/repo/migrations/`:**
- Ecto migrations. File naming: `YYYYMMDD<seq>_<description_in_underscores>.exs`. The date prefix is in the format `20260418NNNNNN`. All migrations use UUIDs via `Foglet.Schema`.

**`priv/ssh/`:**
- Persisted SSH host keys. Must survive deploys to avoid host-key-changed warnings for users. Created by `Foglet.SSH.Supervisor.ensure_system_dir!/0` at boot.

**`vendor/raxol/`:**
- Raxol terminal UI library included as a `path:` dependency (`{:raxol, path: "vendor/raxol"}` in `mix.exs`). Not fetched from Hex вЂ” the project maintains its own copy so TUI internals can be referenced and patched.

**`test/`:**
- Mirrors `lib/foglet_bbs/` exactly: `test/foglet_bbs/boards/` tests `lib/foglet_bbs/boards/`, and so on.
- `test/support/` holds fixtures, factories, and shared test helpers. TUI test helpers live at `test/support/foglet/tui/`.

## Naming Conventions

**Files:**
- `snake_case.ex` for all Elixir source files.
- Context modules use a flat name matching the domain (e.g. `boards.ex`, `accounts.ex`).
- Schema modules use a flat entity name (e.g. `user.ex`, `post.ex`, `read_pointer.ex`).
- GenServer/Supervisor modules named by role: `server.ex`, `supervisor.ex`.
- TUI screen modules named by screen: `login.ex`, `board_list.ex`, `post_reader.ex`.
- Screen state companion: `<screen_name>/state.ex`.
- Widget modules named by component: `screen_frame.ex`, `smart_list.ex`, `markdown_body.ex`.

**Modules:**
- Application core: `Foglet.<Domain>` (e.g. `Foglet.Accounts`, `Foglet.Boards`).
- Ecto schemas: `Foglet.<Domain>.<Entity>` (e.g. `Foglet.Accounts.User`, `Foglet.Posts.Post`).
- OTP workers: `Foglet.<Domain>.<Role>` (e.g. `Foglet.Boards.Server`, `Foglet.Sessions.Supervisor`).
- TUI app/screens: `Foglet.TUI.App`, `Foglet.TUI.Screens.<ScreenName>`.
- TUI widgets: `Foglet.TUI.Widgets.<Category>.<Name>` (e.g. `Foglet.TUI.Widgets.Chrome.ScreenFrame`) or `Foglet.TUI.Widgets.<Name>` for top-level widgets.
- SSH layer: `Foglet.SSH.<Role>` (e.g. `Foglet.SSH.CLIHandler`).
- Web layer: `FogletBbsWeb.<Role>` (e.g. `FogletBbsWeb.Endpoint`, `FogletBbsWeb.Telemetry`).
- Mix tasks: `Mix.Tasks.Foglet.<Task>` (e.g. `Mix.Tasks.Foglet.User.Create`).

**Tests:**
- Test files named `<source_file>_test.exs` co-located under `test/` at the same relative path as their source under `lib/`.

## Module Organization Pattern

Domains follow a **context + schemas** split:

```
lib/foglet_bbs/
  boards.ex           в†ђ public API (all public functions here)
  boards/
    board.ex          в†ђ Ecto schema + changeset
    category.ex       в†ђ Ecto schema + changeset
    read_pointer.ex   в†ђ Ecto schema + changeset
    subscription.ex   в†ђ Ecto schema + changeset
    server.ex         в†ђ GenServer (implementation detail, called by context)
    supervisor.ex     в†ђ DynamicSupervisor
```

Callers always go through the context module (e.g. `Foglet.Boards.create_board/2`), never directly to `Foglet.Boards.Board` or `Foglet.Boards.Server` unless they are also within the `Foglet.Boards` namespace.

## Where to Add New Code

**New domain feature (e.g. direct messages):**
- Context module: `lib/foglet_bbs/messages.ex`
- Ecto schemas: `lib/foglet_bbs/messages/<entity>.ex`
- Migration: `priv/repo/migrations/<timestamp>_create_<table>.exs` (run `mix ecto.gen.migration`)
- Tests: `test/foglet_bbs/messages/<entity>_test.exs`

**New TUI screen:**
- Screen module: `lib/foglet_bbs/tui/screens/<screen_name>.ex` implementing `Foglet.TUI.Screen`
- If the screen has non-trivial state: `lib/foglet_bbs/tui/screens/<screen_name>/state.ex`
- Register in `TUI.App.screen_module_for/1` and add atom to the `@type screen` type union
- Add screen atom to `TUI.App.build_pubsub_topics/1` if PubSub subscriptions are needed
- Tests: `test/foglet_bbs/tui/screens/<screen_name>_test.exs`

**New TUI widget:**
- Place in the appropriate `lib/foglet_bbs/tui/widgets/<category>/` subdirectory
- Use `Foglet.TUI.Theme.t()` as the theme parameter type; receive it as an argument
- Tests: `test/foglet_bbs/tui/widgets/<category>/<widget>_test.exs`

**New GenServer worker in an existing domain:**
- Implementation: `lib/foglet_bbs/<domain>/<worker_name>.ex`
- Register it in the domain's `Supervisor` (if dynamic) or add to `application.ex` supervision tree (if static)

**New Mix task:**
- File: `lib/mix/tasks/foglet.<task_name>.ex`
- Module: `Mix.Tasks.Foglet.<TaskName>`

**New runtime config key:**
- Add spec to `@entries` in `lib/foglet_bbs/config/schema.ex`
- Add seed row to `priv/repo/seeds.exs`
- Add typed accessor to `lib/foglet_bbs/config.ex`

## Special Directories

**`vendor/raxol/`:**
- Purpose: Local copy of the Raxol TUI library, included as `{:raxol, path: "vendor/raxol"}`.
- Generated: No вЂ” maintained manually.
- Committed: Yes вЂ” the full vendored source is in the repo.
- Docs for Raxol are in `docs/raxol/`; always consult `docs/raxol/README.md` before writing TUI code.

**`priv/ssh/`:**
- Purpose: SSH host key storage. Keys are generated by the Erlang `:ssh` daemon on first boot.
- Generated: Yes (by `:ssh` at runtime).
- Committed: No вЂ” keys are deploy-specific secrets.

**`_build/`:**
- Purpose: Elixir/Mix compile output.
- Generated: Yes.
- Committed: No.

**`.planning/`:**
- Purpose: GSD planning artifacts (requirements, ADRs, phase plans, verification checklists). Read before proposing changes to existing decisions.
- Generated: By GSD commands.
- Committed: Yes.
