## Critical Gaps

### 1. Unsupported reset-password URL

Affected requirements: MAIL-04, MAIL-06, HYGN-02, HYGN-03.

`mix foglet.user.reset_password HANDLE` prints `https://<host>/users/reset_password/<token>`, and README documents that URL as the break-glass reset workflow. The Phoenix router has no supported reset route, and the SSH/TUI product has no reset-token consumption path. This makes the reset retrieval path a false launch promise.

Evidence:

- `lib/mix/tasks/foglet.user.reset_password.ex`
- `lib/foglet_bbs_web/router.ex`
- `README.md`
- `test/mix/tasks/foglet_user_reset_password_test.exs`
- `.planning/phases/14-launch-hygiene-and-operator-notes/14-BLOCKERS.md`

### 2. README contradicts blocker log

Affected requirement: HYGN-03.

README says no upstream Phase 9-13 blockers are currently recorded, while `14-BLOCKERS.md` records the reset URL blocker as Phase 9 / Phase 14. Operator notes are therefore not reliable enough for pre-alpha completion.
