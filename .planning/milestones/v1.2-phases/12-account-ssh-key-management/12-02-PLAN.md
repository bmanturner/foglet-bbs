<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Terminal input -> Account screen | Label, public-key text, and action key presses enter screen-local state. |
| Account screen -> Accounts | TUI delegates key list/add/revoke operations to context APIs. |
| Account state -> Renderer | Stored key metadata becomes terminal output. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-12-06 | Elevation of Privilege | `SSHKeysActions.revoke_selected/2` | mitigate | Call only `Accounts.revoke_ssh_key(actor, key.id)`; ownership is enforced in Accounts from Plan 01. |
| T-12-07 | Information Disclosure | `SSHKeysSurface.render/2` | mitigate | Render label, fingerprint, created time, and last-used state; do not render raw `public_key`. |
| T-12-08 | Tampering | add flow attrs | mitigate | Send only `%{label: label, public_key: public_key}` to `Accounts.register_ssh_key/2`; user_id remains context-set. |
| T-12-09 | Denial of Service | invalid form submissions | mitigate | Invalid changesets become inline error strings; screen remains usable and does not crash. |
| T-12-10 | Spoofing | tab visibility | accept | SSH KEYS tab is visible to authenticated users only because Account requires `current_user`; persistence authorization remains in Accounts. |
</threat_model>
