<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| unauthenticated TUI -> Accounts | Anyone at Login can submit a reset identifier. |
| Accounts -> token table | Matching active users get reset tokens persisted. |
| Accounts -> Swoosh | Reset instructions leave Foglet only in email mode. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-07 | Information Disclosure | `request_password_reset_delivery/1` | mitigate | Always return `{:ok, :generic_response}` in email mode for known, unknown, deleted, inactive, and delivery-failed cases. |
| T-09-08 | Information Disclosure | `Login` reset copy | mitigate | Use one generic success string and forbid browser reset route strings. |
| T-09-09 | Tampering | `Login` reset availability | mitigate | Gate reset subflow on `Foglet.Config.delivery_mode() == "email"` and return unavailable in no-email mode. |
</threat_model>
