<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| operator runtime config -> DB config | Non-secret delivery mode crosses from sysop-controlled config into persisted runtime rows. |
| runtime env -> mailer adapter | SMTP/API credentials remain in environment/OTP config and must not cross into DB config. |
| Accounts email builders -> external email provider | Verification and reset contents leave the system through Swoosh. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-01 | Information Disclosure | `Foglet.Config.Schema` | mitigate | Persist only `"delivery_mode"` enum values `"email"` and `"no_email"`; reject provider/credential keys in DB-backed config per D-03. |
| T-09-02 | Spoofing | `Foglet.Mailer` | mitigate | Use one OTP-configured `Foglet.Mailer` boundary; provider credentials come from runtime env and are never accepted from TUI or DB config. |
| T-09-03 | Information Disclosure | `Foglet.Accounts.Email.password_reset/2` | mitigate | Do not include browser reset URLs or raw reset tokens in user-facing TUI copy; email builders must avoid `/users/reset_password/` routes. |
</threat_model>
