<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| TUI/Mix -> Accounts | Operator input crosses into authoritative account mutation APIs. |
| Accounts -> Postgres | Status state is persisted and must remain in the locked graph. |
| Actor -> Authorization | User-provided/session actor identity is tested for sysop authority. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-01 | Elevation of Privilege | `Accounts.transition_user_status/3` | mitigate | Call `Bodyguard.permit/4` with `:manage_user_status` before target lookup/mutation side effects. |
| T-10-02 | Tampering | status transition graph | mitigate | Permit only `pending -> active`, `pending -> rejected`, `active -> suspended`, and `suspended -> active`; tests assert invalid transitions preserve DB status. |
| T-10-03 | Tampering | deleted targets | mitigate | Return `{:error, :deleted}` for non-nil `deleted_at` before `Repo.update/1`. |
| T-10-04 | Elevation of Privilege | rejected actor scopes | mitigate | Add rejected actor guard in `authorize/3` and `scopes_for/2`. |
| T-10-05 | Repudiation | status API return values | mitigate | Return tagged atoms/maps so callers do not parse text to distinguish success/failure. |
</threat_model>
