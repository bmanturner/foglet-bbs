## Accepted Requirement Adjustment

### MENU-01

**Requirement:** User sees current date and time in the top-right screen chrome rendered in their saved timezone and 12h/24h preference, defaulting to system timezone and 12-hour time when no user preference has been saved yet.

**Decision:**

The user clarified on 2026-04-24 that only the time matters in the chrome and that the date was intentionally removed. The milestone is therefore accepted with time-only display as the intended behavior.

**Evidence retained for traceability:**

- `06-VERIFICATION.md` status is `gaps_found`.
- `ClockFormatter.format/2` converts timezone and honors 12h/24h, but returns time-only strings.
- `status_bar_test.exs` explicitly refutes `"2026-04-24"`, so tests currently protect the opposite of the date requirement.
- Integration checker confirmed the Phase 5 preference data path is wired; timezone and `time_format` are honored.

**Follow-up:**
