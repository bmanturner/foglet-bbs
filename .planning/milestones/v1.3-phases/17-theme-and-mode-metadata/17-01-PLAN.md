<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| TUI route metadata | `current_screen` metadata crosses into future presentation decisions. It must not become an authorization or permission boundary. |
| Screen id lookup | Caller-supplied or future screen ids enter `mode_for!/1`; unknown ids must not default to a valid mode. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-17-01 | Tampering | `Foglet.TUI.Presentation.mode_for!/1` | mitigate | Reject unknown screen ids by raising `ArgumentError` containing `unknown TUI screen`; add ExUnit coverage in `presentation_test.exs`. |
| T-17-02 | Elevation of Privilege | Presentation mode metadata | mitigate | Document in `Foglet.TUI.Presentation` and tests that mode is display metadata only; do not consult `Foglet.Authorization` or use mode for access decisions. |
| T-17-03 | Information Disclosure | Operator mode labels | accept | Mode values are coarse product presentation labels and expose no user data, secrets, or authorization scopes. |
</threat_model>
