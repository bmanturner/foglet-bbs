| Boundary | Description |
|----------|-------------|
| user input -> TUI layout helper | Untrusted Unicode text is measured, truncated, padded, or sliced for terminal display. |
| TUI layout helper -> product validation | Display-width results must not become authorization, verification-code, title-length, or post-body validation policy. |

| T-16-01 | T/I | `Foglet.TUI.TextWidth.truncate/3` | mitigate | Implement truncation only for display output; tests cover combining marks and CJK so output does not split visual glyphs or leak overflow columns. |
| T-16-02 | E | `Foglet.TUI.TextWidth.display_width/1` consumers | mitigate | Keep helper in `Foglet.TUI` and document it as terminal layout infrastructure, not validation or authorization. Later plans preserve character-count policies per D-07/D-08. |
| T-16-03 | D | padding/truncation helpers | mitigate | Clamp non-positive widths and negative padding so hostile input or tiny terminal widths cannot crash render paths. |
