<interfaces>
`Foglet.TUI.Presentation.theme_mappings().editor` exposes `%{focused: :accent, unfocused: :border, input: :primary, counter: :dim, counter_warning: :warning, counter_error: :error}`.

`Foglet.TUI.Widgets.Compose.render_input/4` returns the existing width-aware `MultiLineInput` render tree. EditorFrame must accept this as a child and must not call `MultiLineInput.update/2`.

Use a stateless public API such as:

```elixir
EditorFrame.render(
  mode: :edit | :preview,
  focused?: boolean(),
  context: [render_tree()] | render_tree() | nil,
  title: render_tree() | nil,
  body: render_tree(),
  budgets: [%{label: String.t(), count: non_neg_integer(), limit: pos_integer()}],
  error: String.t() | nil,
  width: pos_integer(),
  height: pos_integer(),
  theme: %Foglet.TUI.Theme{}
)
```

Exact option names may differ, but both screens must be able to pass pre-rendered title/context/body/preview content and budget maps through one shared widget.
</interfaces>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| TUI render state -> terminal output | Untrusted draft text is displayed in shell/counter/preview surfaces, but no mutation occurs in the widget. |
| Widget -> domain contexts | No crossing allowed; EditorFrame must not call `Foglet.Posts`, `Foglet.Threads`, or authorization APIs. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-23-01 | Tampering | `EditorFrame` counters | mitigate | Counters are display-only; widget receives count/limit and never authorizes or validates submit. |
| T-23-02 | Elevation of Privilege | `EditorFrame.render/1` | mitigate | Widget creates no submit path and no domain mutation API. |
| T-23-03 | Tampering | Input handling boundary | mitigate | EditorFrame accepts pre-rendered children and does not call `Compose.translate_key/1` or `MultiLineInput.update/2`. |
| T-23-04 | Denial of Service | Narrow composer layout | mitigate | Widget exposes compact text counters and in-shell mode labels that later layout smoke tests can assert at 64x22. |
</threat_model>
