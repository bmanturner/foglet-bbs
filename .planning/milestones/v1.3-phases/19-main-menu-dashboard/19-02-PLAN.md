<panel_width_budget>
The Navigation panel inner width is derived from `state.terminal_size` using the same conservative `split_pane(direction: :horizontal, ratio: {2, 3})` math the body uses, then floored at a minimum to guarantee 64x22 fit:

```
outer_width    = elem(state.terminal_size, 0)             # full terminal width
chrome_outer   = 4                                         # ScreenFrame outer border + padding
left_alloc     = (outer_width - chrome_outer) * 2 div 5    # split_pane ratio {2,3}
box_border     = 2                                         # `box border: :single` left + right
inner_width    = max(left_alloc - box_border, @nav_panel_min_inner_width)
```

Worked examples:
- `{64, 22}` → `(64 - 4) * 2 / 5 - 2 = 22` → floor at `@nav_panel_min_inner_width` (20) is irrelevant; `22` wins.
- `{80, 24}` → `(80 - 4) * 2 / 5 - 2 = 28` → `28` wins.
- `{132, 50}` → `(132 - 4) * 2 / 5 - 2 = 49` → `49` wins.

The minimum constant `@nav_panel_min_inner_width 20` exists so `nav_row/3` never produces negative padding when `state.terminal_size` is unset (e.g. in test fixtures that don't set it) or pathologically small. A row like `● Boards` (8 cells) + padding + `B` (1 cell) = 20 cells fits at the floor. The longest label `Moderation` (10 cells) gives `⚑ Moderation` (12 cells) + padding + `M` (1 cell) = 13 cells used + 7 cells padding — also fits at the floor.
