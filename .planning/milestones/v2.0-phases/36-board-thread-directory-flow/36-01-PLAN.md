Implement these reducer messages:
- `:load` returns the current state with `status: :loading`, `last_op: :load_boards`, and one `Effect.task(:load_boards, :board_list, fun)`.
- `{:task_result, :load_boards, {:ok, directory}}` stores `directory`, sets `status` to `:empty` when `directory == []` and `:loaded` otherwise, clears `last_error`, and creates or refreshes `board_tree` with `BoardTree.init(directory: directory, id: "board-directory")`.
- `{:task_result, :load_boards, {:error, reason}}` stores `status: {:error, reason}`, clears `last_op`, and preserves any existing `directory` and `board_tree`.
- `{:key, %{key: :down}}`, `{:key, %{key: :up}}`, `{:key, %{key: :left}}`, `{:key, %{key: :right}}`, `{:key, %{key: :char, char: "j"}}`, and `{:key, %{key: :char, char: "k"}}` update only local `board_tree` through `BoardTree.handle_event/2`.
- `{:key, %{key: :enter}}` toggles category expansion when the focused node is a category; when the focused node is a board, it emits `Effect.navigate(:thread_list, %{board: board_identity, board_id: board.id})`.
- `{:key, %{key: :char, char: "s"}}` on an unsubscribed board emits `Effect.task(:subscribe_to_board, :board_list, fun)` and clears feedback.
- `{:key, %{key: :char, char: "u"}}` on a subscribed, non-required board emits `Effect.task(:unsubscribe_from_board, :board_list, fun)` and clears feedback.
- `{:key, %{key: :char, char: "u"}}` on a required-subscription board sets feedback to `"This board is a required subscription."` and emits no effects.
- `{:task_result, :subscribe_to_board, {:ok, _}}` sets feedback to `"Subscribed."` and emits a `:load_boards` task effect.
- `{:task_result, :unsubscribe_from_board, {:ok, _}}` sets feedback to `"Unsubscribed."` and emits a `:load_boards` task effect.
- `{:task_result, op, {:error, :required_subscription}}` for subscription ops sets feedback to `"This board is a required subscription."`.
- `{:task_result, op, {:error, :board_archived}}` for subscription ops sets feedback to `"That board is archived."`.
- `{:task_result, op, {:error, reason}}` for subscription ops sets feedback to `"Subscription change failed: #{inspect(reason)}"`.
- `{:board_activity, board_id, event}` emits a `:load_boards` task effect when `board_id` is binary; it may ignore `event`.
- `{:key, %{key: :char, char: c}}` with c in ["q", "Q"] emits `Effect.navigate(:main_menu, %{})`.

The load task closure must bind only `user = context.current_user` and
`boards_mod` resolved from `context.domain` or `context.session_context` before
creating the closure. The subscribe/unsubscribe task closures must bind only
`user`, `board_id`, and `boards_mod`.

Implement `render/2` so it renders from `%State{}` plus `%Foglet.TUI.Context{}`.
Use `context.terminal_size` for layout, `Theme.from_state/1` through a minimal
render model if necessary, and `State.directory` instead of App `board_list`.
Preserve the existing loading, empty, feedback, BoardTree, and detail-strip
render branches.

After `render/2` and `update/3` are complete, remove the exported legacy
`handle_key/2`, `render/1`, and `load_boards/1` callbacks from BoardList.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| BoardList reducer -> Foglet.Boards | Screen task closures call board directory and subscription APIs. |
| Task result -> BoardList reducer | Async board directory/subscription outcomes update screen-local state. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-36-01 | Elevation of Privilege | Subscribe/unsubscribe task effects | mitigate | Keep authorization and mutation behavior inside `Foglet.Boards`; BoardList only requests context calls and consumes results. |
| T-36-02 | Information Disclosure | Board directory route params | mitigate | Route params carry selected board identity already visible in the directory; no hidden board fields or authorization decisions move into UI. |
| T-36-03 | Denial of Service | Board activity refresh | mitigate | Refresh remains a single BoardList load task per routed activity message, not synchronous Repo work in the update path. |
</threat_model>
