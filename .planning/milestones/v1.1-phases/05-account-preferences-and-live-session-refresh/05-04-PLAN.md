<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Account save command -> Accounts context | Untrusted UI drafts become persisted Account attrs. |
| Accounts result -> active TUI/Session snapshots | Persisted preferences become live display state. |
| failed save -> active snapshots | Invalid values must not partially refresh the UI or Session state. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-12 | Elevation of Privilege | Account save payload | mitigate | Payload contains only private profile and presentation preference keys; Accounts changeset also refuses privileged casts. |
| T-05-13 | Tampering | live refresh | mitigate | Refresh snapshots only after `Accounts.update_profile/2` returns `{:ok, updated_user}`. |
| T-05-14 | Tampering | failed save path | mitigate | On changeset error, keep `current_user`, `session_context`, and Session state unchanged and render errors from the changeset. |
| T-05-15 | Denial of Service | final precommit | mitigate | Run compile, format, Credo, Sobelow, and Dialyzer through `mix precommit`; fix issues instead of bypassing the gate. |
</threat_model>
