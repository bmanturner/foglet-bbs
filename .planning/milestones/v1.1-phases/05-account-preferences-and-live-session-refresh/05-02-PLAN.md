<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| persisted user preferences -> live Session state | Stored values become active runtime display state. |
| TUI code -> Session GenServer | Account save will refresh session through a public API. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-04 | Tampering | `Sessions.Preferences.from_user/1` | mitigate | Resolve only registered theme ids and fall back to `gray`; do not create atoms from strings. |
| T-05-05 | Information Disclosure | `Session.get_state/1` | accept | Snapshot contains display preferences only: timezone, time format, theme id, and resolved theme. |
| T-05-06 | Tampering | `Session.update_preferences/2` | mitigate | Public API updates only preference snapshot fields, not user_id, handle, role, or terminal state. |
| T-05-07 | Repudiation | `CLIHandler.build_context/3` | accept | Startup preference seeding is deterministic from the saved user row; no audit trail required for ASVS L1. |
</threat_model>
