## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| TUI/user input -> Accounts changeset | Untrusted profile and preference fields enter persistence. |
| Theme id string -> theme registry | User-controlled theme strings must not become atoms. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Tampering | `User.profile_changeset/2` | mitigate | Validate timezone with `Timex.Timezone.exists?/1`, time format against `["12h", "24h"]`, theme against registered string ids, and reject invalid changesets. |
| T-05-02 | Elevation of Privilege | `User.profile_changeset/2` | mitigate | Cast only profile/preference fields; never cast handle, email, password, role, status, invite policy, or counters. |
| T-05-03 | Denial of Service | theme validation | mitigate | Do not call `String.to_atom/1`; compare strings from `Atom.to_string/1` over `Theme.ids/0`. |
</threat_model>
