<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| TUI/API caller -> `Foglet.Oneliners.hide_entry/3` | Untrusted actor, target, and reason enter the authoritative domain mutation. |
| `Foglet.Oneliners` -> `Foglet.Authorization` | Authorization scope is checked before persistence. |
| `Foglet.Oneliners` -> `Foglet.Moderation` | Successful mutation writes a durable audit record. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-01 | Spoofing | `Foglet.Oneliners.hide_entry/3` | mitigate | Ignore caller-supplied moderator ids; use `actor` and call `Bodyguard.permit/4` before side effects. |
| T-08-02 | Tampering | `hidden_by_id`, `mod_id`, `target_id` | mitigate | Programmatically set ids from trusted structs; schema changesets do not cast programmatic ids from attrs. |
| T-08-03 | Repudiation | `mod_actions` audit | mitigate | Insert exactly one append-only `:hide_oneliner` action with moderator, target, reason, metadata, and timestamp after successful hide. |
| T-08-04 | Information Disclosure | audit list scope filtering | mitigate | `list_actions_for_scopes/2` returns `[]` for empty scopes and accepts list-shaped `:site | {:board, id}` scopes. |
| T-08-05 | Denial of Service/Layout | reason/body metadata | mitigate | Reason is trimmed and required; TUI rendering in later plans truncates text, while this plan keeps persistence bounded by existing oneliner body limit. |
</threat_model>
