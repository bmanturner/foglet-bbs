## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Sysop tab visibility -> operator actions | Sysop tab exposure must track runtime policy but is not authorization. |
| Sysop screen -> shared actions | Sysop key input crosses into shared action code, then into `Foglet.Accounts`. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-11 | E | Sysop.handle_key/2 | mitigate | Delegate invite actions to `InvitesActions`; `Foglet.Accounts` remains authoritative for sysop authorization and unlimited operator generation. |
| T-04-12 | T | Sysop.State tabs | mitigate | Recompute `ShellVisibility.invites_visible?/2` from runtime policy and clamp active tab if INVITES disappears. |
| T-04-13 | S | Sysop current_user handling | mitigate | Nil and non-sysop users remain blocked by `ShellVisibility.sysop_visible?/1`; no invite state action is reachable from unauthorized Sysop render. |
| T-04-14 | R | Sysop revoke display | mitigate | Shared actions refresh after successful revoke and preserve error state after failed revoke, preventing spoofed success. |
</threat_model>
