## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Account tab visibility -> user input | Hidden Account tab is advisory and must not be treated as authorization. |
| Account screen -> shared actions | Account hands untrusted key input and current actor to shared invite behavior. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-05 | E | Account.handle_key/2 | mitigate | Delegate active INVITES actions to `InvitesActions`, which calls `Foglet.Accounts`; Account must contain no direct invite mutation calls. |
| T-04-06 | T | Account tab state | mitigate | Recompute `ShellVisibility.invites_visible?/2` from runtime `session_context` and rebuild tabs when policy changes to avoid stale Account exposure. |
| T-04-07 | I | Account INVITES list | mitigate | Account does not query or filter invite records; shared actions consume only `Accounts.list_invites/1`. |
</threat_model>
