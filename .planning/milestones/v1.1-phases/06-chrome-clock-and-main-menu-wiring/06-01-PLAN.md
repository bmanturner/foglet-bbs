<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| session snapshot -> chrome widget | User preference data crosses into render code and may be missing or invalid. |
| role/session context -> menu routing | Refreshed user role and policy state determines visible and accepted navigation. |
| Raxol interval -> App.update/2 | Runtime tick messages can trigger rerender paths without user input. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01 | Elevation of privilege | `Foglet.TUI.Screens.MainMenu` | mitigate | Tests must derive expected Account/Moderation/Sysop render and key behavior from `ShellVisibility` predicates. |
| T-06-02 | Information disclosure / Denial of service | `Foglet.TUI.Widgets.Chrome.StatusBar` | mitigate | Tests must require state/session-snapshot clock inputs and no render-time persistence reads. |
| T-06-03 | Denial of service | `ClockFormatter` / `StatusBar` | mitigate | Tests must require fallback for invalid or missing timezone and time format. |
</threat_model>
