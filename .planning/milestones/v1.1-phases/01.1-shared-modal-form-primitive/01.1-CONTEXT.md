# Phase 01.1: shared-modal-form-primitive - Context

**Gathered:** 2026-04-23 (assumptions mode)
**Status:** Ready for planning

<domain>
## Phase Boundary

Deliver `Foglet.TUI.Widgets.Modal.Form` — a stateful D-14 modal-overlay widget that hosts a caller-declared list of typed input fields (`:text`, `:integer`, `:boolean`, `:enum`, `:textarea`), supports wrap-around Tab/Shift-Tab focus navigation, submits via `on_submit/1` with a `%{field_name => typed_value}` map, cancels via `on_cancel/0` on Esc, and renders caller-supplied per-field inline errors without dismissing. Scope is the primitive only — no rewiring of existing screens (`NewThread`, etc.); adoption happens in Phase 2+ as CRUD flows land.
</domain>

<decisions>
## Implementation Decisions

### Module Structure
- **D-01:** New module `Foglet.TUI.Widgets.Modal.Form` at `lib/foglet_bbs/tui/widgets/modal/form.ex` (sibling subdir of `modal.ex`). Single module per file per project CLAUDE.md.
- **D-02:** Pure D-14 stateful widget triplet (`init/1 + handle_event/2 + render/2`). No process. State lives in a struct held by the caller.
- **D-03:** State struct fields: `title`, `fields` (FieldSpec list), `field_states` (parallel state list), `focus_index :: non_neg_integer()`, `errors :: %{atom() => String.t()}`, `on_submit :: (map() -> any())`, `on_cancel :: (-> any())`.
- **D-04:** `handle_event/2` returns `{new_state, action}` where action is one of `:submitted | :cancelled | nil`. Matches TextInput contract.
- **D-05:** `modal.ex` is not edited. REQ-8 byte-identity is enforced by review.

### Field Type Adapters
- **D-06:** `:text` and `:integer` fields use `Foglet.TUI.Widgets.Input.TextInput` as a stateful child (its FieldState wraps a TextInput state).
- **D-07:** `:boolean` uses `Checkbox.render/2` (D-16 stateless). FieldState holds raw `boolean()`. Toggle on Space/Enter while focused.
- **D-08:** `:enum` uses `RadioGroup.render/3` (D-16 stateless). FieldState holds raw `selected_index :: integer()`. Up/Down arrow keys adjust index when focused.
- **D-09:** `:textarea` uses `Raxol.UI.Components.Input.MultiLineInput` directly, with `Foglet.TUI.Widgets.Compose.translate_key/1` for key translation. FieldState wraps the MLI state.
- **D-10:** Reuse existing primitives — no hand-rolled buffers, cursors, or radio logic. New code is the container, the dispatch, and the textarea adapter only.

### Focus Navigation
- **D-11:** Focus is tracked as `focus_index :: non_neg_integer()` (integer, not atom) so the form generalizes to N fields.
- **D-12:** Tab advance: `rem(focus_index + 1, n)`. Shift-Tab retreat: `rem(focus_index - 1 + n, n)`. Wrap both directions per REQ-4.
- **D-13:** Event interception order in `handle_event/2`: (1) Esc → `on_cancel`; (2) Tab → advance; (3) Shift-Tab → retreat; (4) Enter on last index → coerce + `on_submit`; (5) Enter on non-last → behave as Tab; (6) all else → dispatch to focused field. Tab/Esc must be intercepted BEFORE child dispatch (per RESEARCH Pitfall 1).
- **D-14:** Shift-Tab is matched as `%{key: :tab, shift: true}` based on analogy with `%{key: :char, ctrl: true}` used in `lib/foglet_bbs/tui/screens/new_thread.ex:237`. **Risk:** if Raxol's actual shape differs, the guard never matches. **Mitigation:** the Shift-Tab test (REQ-4 reverse-wrap) MUST be written against a real keystroke fixture first; the guard pattern is whatever shape the test discovers. This is the single Likely-not-Confident decision in the phase.

### Coercion (per REQ-3)
- **D-15:** Coercion happens at submit time in `handle_event/2` via a `collect_values/1` helper, not on each keystroke.
- **D-16:** `:integer` coercion uses `Integer.parse(String.trim(text))` returning `nil` on `:error` or trailing-garbage. Spec REQ-3 explicitly permits `integer() | nil` so the caller's validator can reject blanks.
- **D-17:** `:text` and `:textarea` deliver `String.t()` (textarea may contain `\n`). `:boolean` delivers the raw boolean. `:enum` delivers the choice atom/value at `selected_index` (the original choice declared in the FieldSpec, not the index).

### Errors API (per REQ-7)
- **D-18:** Pure setter `set_errors/2 :: (t(), %{atom() => String.t()}) :: t()`. Returns updated state with `errors` replaced. The form does not validate; the caller orchestrates the set/clear/dismiss cycle.
- **D-19:** `:ok` from `on_submit` is the caller's signal to dismiss. The form itself never auto-dismisses on submit. `set_errors/2` with `%{}` clears errors.
- **D-20:** Inline error rendering: a single `text/2` line beneath the field widget in `theme.error.fg`. Mirrors `new_thread.ex:119`.

### Render
- **D-21:** `render/2` returns a bare `column` (title row + divider + per-field rows + key-hint row). NO outer `box`/border. The chrome comes from `Foglet.TUI.App.render_modal_overlay/2` (per RESEARCH Pitfall 4 to avoid double border).
- **D-22:** Focused field label uses `theme.accent.fg` + `[:bold]`; unfocused uses `theme.primary.fg` + `[]`. Mirrors NewThread's title_label pattern (`new_thread.ex:101-103`).
- **D-23:** Key hint row at the bottom: `[Tab] Next  [Shift+Tab] Prev  [Enter] Submit  [Esc] Cancel` in `theme.dim.fg`.
- **D-24:** All color decisions route through `Foglet.TUI.Theme` slots — no hardcoded color atoms (D-07/D-09 enforced by `color_atom_leaked?/2` in tests).

### `@moduledoc`
- **D-25:** Module doc cites D-07, D-09, D-13, D-14 per SPEC REQ-1.

### Testing
- **D-26:** Single test file `test/foglet_bbs/tui/widgets/modal/form_test.exs` covering all 9 reqs: smoke render, theme hygiene (D-18), 5-field-type render, Tab wrap (`0→1→2→0`), Shift-Tab wrap (`0→2→1→0`), Enter-on-last-submits-once, Esc-cancels-once, `set_errors/2` renders inline + modal stays open, E2E open→navigate→submit fixture.
- **D-27:** Use `ExUnit.Case, async: true`. Import `Foglet.TUI.WidgetHelpers` for `flatten_text/1`, `color_atom_leaked?/2`, `color_names/0`. Mirror structure of `test/foglet_bbs/tui/widgets/input/text_input_test.exs`.
- **D-28:** REQ-8 regression: `test/foglet_bbs/tui/widgets/modal_test.exs` continues to pass unchanged; verified by re-running it under `mix test`.

### Acceptance / Quality Gate
- **D-29:** `mix precommit` (compile --warnings-as-errors, format, credo --strict, sobelow, dialyzer) must pass before Phase 01.1 is considered done.
- **D-30:** Update `lib/foglet_bbs/tui/widgets/README.md` to add a `Modal.Form` entry under the Compose / Modal section.

### Claude's Discretion
- Naming of `FieldSpec` and `FieldState` internal structs (or whether to use bare maps). Recommend bare keyword/map for FieldSpec (caller-declared) and a small struct for FieldState (internal mutable state).
- Whether `on_submit` is invoked synchronously inline in `handle_event/2` (recommended, matches existing TextInput.on_submit pattern) vs returned in the action tuple. Prefer inline call for symmetry with REQ-5/6.
- Choice of whether `:textarea` height is fixed at field-spec time (`:rows`) or dynamic. Recommend fixed `:rows` per FieldSpec, defaulting to 3.

<code_context>
## Existing Code Insights

### Reusable Assets
- `Foglet.TUI.Widgets.Input.TextInput` — full D-14 widget; handles cursor/backspace/mask/max_length and `:submitted` action on Enter
- `Foglet.TUI.Widgets.Input.Checkbox` — `render/2` only; theme-routed; caller stores boolean
- `Foglet.TUI.Widgets.Input.RadioGroup` — `render/3` only; theme-routed with `clamp_index`; caller stores selected_index
- `Raxol.UI.Components.Input.MultiLineInput` — full multi-line buffer with cursor + undo/redo; consumed by `NewThread` and `PostComposer`
- `Foglet.TUI.Widgets.Compose.translate_key/1` — translates Foglet key events into MultiLineInput messages; returns `nil` for Tab/Esc so they bubble to the form level
- `Foglet.TUI.App.render_modal_overlay/2` — provides the `:double` border + centering chrome
- `Foglet.TUI.Theme` — all color slots (`primary`, `accent`, `error`, `dim`, `title`)
- `Foglet.TUI.WidgetHelpers` (test/support) — `flatten_text/1`, `color_atom_leaked?/2`, `color_names/0`

### Established Patterns
- D-14 stateful widget triplet (`init/1 + handle_event/2 + render/2`) returning `{state, action_or_nil}`
- D-16 stateless widget pattern (`render` only — caller holds state)
- D-13 theme routing: `theme:` always passed as a render kwarg via `Keyword.fetch!/2`
- D-07/D-09 theme-color hygiene enforced by `color_atom_leaked?/2` in widget tests
- Focus tracked as a single field on the container's struct; events for the focused child dispatched to that child's sub-state
- Per-field render: label (focused-fg + bold or primary-fg) → field widget → optional error row in `theme.error.fg`
- Modal widgets render only the inner body; chrome lives in `App.render_modal_overlay/2`
- Modifier keys arrive as flat fields on the event map (`%{key: :char, ctrl: true}` per `new_thread.ex:237`)

### Integration Points
- Modal.Form is invoked by future Phase 2 screens (INVT-06/07, SYSO-02/03/04) — the screens hold the `%Modal.Form{}` state in their own state struct, dispatch `handle_event/2`, and render `Modal.Form.render(state, theme: theme)` inside the `App.render_modal_overlay/2` slot already used for the existing Modal variants.
- No edits to `Foglet.TUI.App` are required: the overlay slot accepts any view tree, and the existing modal flow already routes events to the active modal first.
</code_context>

<specifics>
## Specific Ideas

- The Shift-Tab event-shape uncertainty (D-14) is the only Likely-not-Confident decision. The mitigation is procedural: the planner should sequence the Shift-Tab test BEFORE the Shift-Tab handler, so the actual event shape is discovered empirically and the guard pattern is written to match. RESEARCH Open Question 1 lists this as the one assumed claim out of all primary citations.
- `:enum` value delivery: deliver the original choice value (atom or string from FieldSpec `:choices`), not the integer index. Reduces caller boilerplate (`Enum.at(choices, idx)`).
- Default `:textarea` `:rows` to 3 unless the FieldSpec overrides.
</specifics>
