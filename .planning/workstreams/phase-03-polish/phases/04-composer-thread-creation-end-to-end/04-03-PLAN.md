# Plan 03 — Composer Wiring: Routing, Origin, Cancel, Submit, Title Cap, Reply-Jump

<objective>
Make the three composer entry points (`[C]` from ThreadList, `[C]` from MainMenu, `[R]` from PostReader) route to the right screen, land the user on the right screen after submit or cancel, and delete the broken `thread_list.ex:80-82` branch that the ROADMAP calls out as the crash path. Also enforce the new soft title cap seeded by Plan 04-02 (D-13, D-14) and implement the reply-jump command extension (D-05) that makes `[R]` → submit land the user on their new reply, not the one they replied to.

This plan consolidates the eight behavior changes that span multiple files so that a single commit — or a short sequence of task commits — ships the COMPOSE-01/02/03 success criteria end-to-end. Plan 04-04 is the cosmetic cleanup (preview integration, dead-code removal) and lands after this plan in the same wave.

      length(posts) - 1
    else
      ss.selected_post_index
    end
