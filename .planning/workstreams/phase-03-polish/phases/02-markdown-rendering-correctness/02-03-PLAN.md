# Plan 03 — PostReader Integration: PostCard + j/k Scroll + Per-Screen Render Cache

<objective>
Replace `PostReader`'s legacy `render_markdown_tuples/2` function and manual per-post assembly with a single `Post.PostCard.render_from_tuples/5` call per displayed post, add j/k within-post scroll keys, extend `screen_state[:post_reader]` with `scroll_offset` and `render_cache`, and reset `scroll_offset` to 0 on N/P post navigation (D-03, D-04, D-05). The render cache memoizes `Foglet.Markdown.render/1` output keyed on `{post.id, width}` (posts are immutable in v1.0.1 so no `updated_at` is needed); on terminal resize the `width` component of the key automatically evicts stale cache entries, which is the correct behavior for RENDER-02. The legacy `render_markdown_tuples/2` function is deleted from `post_reader.ex` (composer/new-thread copies stay untouched — their preview pipeline is not in Phase 2 scope; see CONTEXT.md Phase Boundary).

**Scroll key choices (D-04, locked in DISCUSSION-LOG):** `j` scrolls down one line, `k` scrolls up one line; N/P navigation resets `scroll_offset` to 0. No Space/b page-scroll in Phase 2 (can be added later without API churn).

**Cache location (Claude's Discretion — memoization):** Cache lives in `state.screen_state[:post_reader][:render_cache]` keyed on `{post.id, width}`. Lookup and population happen in a single helper (`get_or_parse_tuples/3`). The cache is rebuilt on every re-entry to PostReader because `screen_state[:post_reader]` is deleted on `Q` (see `post_reader.ex:87`) — this is the per-screen memoization described by ARCHITECTURE.md §2.4 and CONTEXT.md.

**Available body height (for RENDER-02 scroll bounds):** `available_height = max(h - 10, 5)` where `h` is the terminal rows. The `10` accounts for chrome overhead: outer border (2) + status bar (1) + top divider (1) + bottom KeyBar (2) + PostCard header (3: two dim lines + divider) + 1 safety margin. The `max(_, 5)` floor prevents scroll bounds from collapsing on tiny terminals; Phase 5 will introduce the proper "too small" gate.
</objective>

```elixir
defp render_post_content(state, ss, theme, w, h) do
  posts = state.posts
  total = length(posts)
  idx = ss.selected_post_index

  if idx >= total do
    column style: %{gap: 0} do
      [text("No more posts.", fg: theme.warning.fg)]
    end

    post = Enum.at(posts, idx)
    available_height = max(h - 10, 5)

      index: idx,
      total: total,
      scroll_offset: ss.scroll_offset,
      max_lines: available_height
    )
  end
```

### 8. Add screen-state helpers

Add these private helpers near the bottom of the module (after `defp advance_post/2` and before `defp render_post_items/4` in the original — or, since that function is deleted, just above `build_flush_context/1`):

```elixir
# Default screen_state shape for post_reader.
# `render_cache` is a map keyed on {post_id, width} → rendered tuples.
defp default_screen_state do
  %{
    selected_post_index: 0,
    scroll_offset: 0,
    render_cache: %{}
  }
end

# Merges the default shape over any existing :post_reader entry so
# legacy states (from pre-Phase-2 sessions mid-flight) get the new
# keys without overwriting selected_post_index.
defp get_screen_state(state) do
  existing = get_in(state.screen_state, [:post_reader]) || %{}
  Map.merge(default_screen_state(), existing)
end

# Cache-aware Foglet.Markdown.render/1 wrapper. Returns the parsed
# tuple list and (if a new parse was performed) the *caller* is
# responsible for writing the result into screen_state.render_cache.
# This function intentionally does NOT mutate state — PostReader is
# pure during render.
defp parse_body(state, post) do
  sc = Map.get(state, :session_context) || %{}
  markdown_mod = get_in(sc, [:domain, :markdown]) || Foglet.Markdown
  body = Map.get(post, :body) || ""

  if function_exported?(markdown_mod, :render, 1) do
    markdown_mod.render(body)
  else
    # Defensive fallback — mirrors the old render_post_items/4 behavior.
    [{body, :plain}]
  end
end

# Populates the render_cache for the currently-selected post if it's
# not yet cached. Called by handle_key/2 clauses that change selection
# or scroll so the next render/1 call has a warm cache. Returns the
# updated screen_state map.
defp warm_cache(ss, state, post, w) do
  key = {post.id, w}

  if Map.has_key?(ss.render_cache, key) do
    ss
  else
    tuples = parse_body(state, post)
    %{ss | render_cache: Map.put(ss.render_cache, key, tuples)}
  end
end
```

### 9. Rewrite `advance_post/2` to reset `scroll_offset` and warm the cache

Replace the existing `advance_post/2` function with:

```elixir
defp advance_post(state, delta) do
  posts = state.posts || []

  if posts == [] do
    :no_match
  else
    ss = get_screen_state(state)
    new_idx = (ss.selected_post_index + delta) |> max(0) |> min(length(posts) - 1)
    post = Enum.at(posts, new_idx)
    {w, _h} = state.terminal_size || {80, 24}

    # D-04: N/P resets scroll_offset to 0.
    ss = %{ss | selected_post_index: new_idx, scroll_offset: 0}
    ss = if post, do: warm_cache(ss, state, post, w), else: ss

    # Local read-pointer advance — flushed on screen transition.
    new_rp =
      if post && state.current_thread do
        Map.put(state.read_position, state.current_thread.id, %{
          last_read_post_id: post.id,
          last_read_message_number: Map.get(post, :message_number, 0)
        })
      else
        state.read_position
      end

    new_screen_state = Map.put(state.screen_state, :post_reader, ss)

    {:update, %{state | screen_state: new_screen_state, read_position: new_rp}, []}
  end
end
```

```elixir
defp scroll_post(state, delta) do
  posts = state.posts || []

  if posts == [] do
    :no_match
  else
    ss = get_screen_state(state)
    post = Enum.at(posts, ss.selected_post_index)

    if is_nil(post) do
      :no_match
    else
      {w, h} = state.terminal_size || {80, 24}
      available_height = max(h - 10, 5)

      total_lines = Foglet.TUI.Widgets.Post.PostCard.body_line_count(post.body)
      # Max offset: never scroll past (total - available_height).
      # Clamp at 0 for short posts.
      max_offset = max(total_lines - available_height, 0)
      new_offset = (ss.scroll_offset + delta) |> max(0) |> min(max_offset)

      ss = %{ss | scroll_offset: new_offset}
      ss = warm_cache(ss, state, post, w)

      new_screen_state = Map.put(state.screen_state, :post_reader, ss)
      {:update, %{state | screen_state: new_screen_state}, []}
    end
  end
end
```
