# Plan 02 — Route Login + Register via `Accounts.post_login_screen/1` (VERIFY-01)

<objective>
Swap the two TUI call sites that inline-check `confirmed_at == nil` to route through the new `Foglet.Accounts.post_login_screen/1` function from Plan 06-01. Wrap the `Logger.info("[verify] code for @...")` calls at both sites in a compile-time `Mix.env() != :prod` guard so verify codes are NEVER logged in production builds (D-12).

**Why this lives in its own plan:** Plan 06-01 proves `post_login_screen/1` with unit tests before any caller depends on it. If Plan 06-01 fails the checker, Plan 06-02 is never started. This plan's churn is surgical — two screens, two tests — and reverting it is a single `git revert` without touching the Accounts API.

**Contract:**
- **`login.ex` `submit_login/1`:** When `authenticate_by_password/2` returns `{:ok, %{status: :active} = user}`, call `Accounts.post_login_screen(user)` and branch:
  - `:main_menu` → emit `{:promote_session, user}` command (same path as today's confirmed-user branch at line 288).
  - `:verify` → build the verify code, log dev-only, route to `:verify` (current behavior preserved).
- **`register.ex` `submit/2`:** After `Accounts.register_user(data)` returns `{:ok, user}`, call `Accounts.post_login_screen(user)` and branch:
  - `:main_menu` → set `current_user` and `current_screen: :main_menu` directly (no verify_state, no verify code built).
  - `:verify` → build the verify code, log dev-only, route to `:verify` (current behavior preserved).
- **Logger guard:** `Logger.info("[verify] code for @#{user.handle}: #{code}")` at both sites is wrapped in `if Mix.env() != :prod do ... end`. The `:prod` branch is compile-stripped so no Mix runtime call occurs in a release.

**Non-goals:**
- No changes to `verify.ex` (Plan 06-03 handles that).
- No changes to `Foglet.Accounts.post_login_screen/1` (locked by Plan 06-01).
- No new commands or App state — `:promote_session` already routes to `:main_menu` at `app.ex:503`.
- No re-verification on config flip mid-session (D-14, D-15 locked).
</objective>
