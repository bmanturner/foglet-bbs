# Plan 04 — Migrate All 9 Screens to ScreenFrame + Replace Hardcoded Colors

<objective>
Migrate every screen in `lib/foglet_bbs/tui/screens/` to use `Chrome.ScreenFrame.render/4`, replacing the manual `box + column + StatusBar + divider + KeyBar` assembly. Replace all hardcoded `fg: :green`, `fg: :cyan`, `fg: :red`, `fg: :yellow` with theme slot references from `state.session_context.theme`. Wire `SelectionList.render/3` in `board_list.ex` and `thread_list.ex`. Delete the old flat `Widgets.StatusBar` and `Widgets.KeyBar` modules after all callers are updated.
</objective>

<tasks>

<read_first>
- `lib/foglet_bbs/tui/screens/login.ex` — read the full current render/1 function to understand exactly what must be replaced; note all hardcoded colors (`fg: :green`, `fg: :cyan`, `fg: :red`)
- `lib/foglet_bbs/tui/widgets/chrome/screen_frame.ex` — re-read ScreenFrame.render/4 signature: `render(state, title, content_element, key_list)`
- `.planning/workstreams/phase-03-polish/phases/01-widget-foundation-theme-screen-chrome/01-UI-SPEC.md` — re-read the color application rules: `fg: :green` → `theme.primary.fg` (unselected), `fg: :cyan` → `theme.accent.fg`, `fg: :red` → `theme.error.fg`
</read_first>

<action>
Rewrite `login.ex`'s `render/1` function to use ScreenFrame. The outer `box` is now handled by ScreenFrame; the inner `column + content + KeyBar` is replaced by passing `content_element` and `key_list` to ScreenFrame.

Changes:
1. Remove alias to `Foglet.TUI.Widgets.KeyBar` (no longer used directly)
2. Add alias to `Foglet.TUI.Widgets.Chrome.ScreenFrame`
3. Rewrite `render/1`:

```elixir
alias Foglet.TUI.Widgets.Chrome.ScreenFrame

@spec render(map()) :: any()
def render(state) do
  mode = registration_mode(state)
  sub = sub_state(state)
  theme = get_in(state, [:session_context, :theme]) || Foglet.TUI.Theme.default()

  content =
    column style: %{gap: 0} do
      [
        case sub do
          :login_form -> render_login_form(state, theme)
          _ -> render_menu(mode, theme)
        end
      ]
    end

  ScreenFrame.render(state, "Login", content, keys_for(sub, mode))
end
```

4. Update `render_menu/2` to accept `theme` as second arg and replace hardcoded colors:
   - `fg: :green` → `fg: theme.primary.fg`

5. Update `render_login_form/2` to accept `theme` as second arg and replace hardcoded colors:
   - `fg: :green` → `fg: theme.primary.fg` (label text)
   - `fg: :cyan` → `fg: theme.accent.fg` (focused input)
   - `fg: :red` → `fg: theme.error.fg` (error text)
   - The `input_fg/1` private function: return `theme.accent.fg` when focused, `theme.primary.fg` when not. Signature changes to `input_fg(focused?, theme)`.

Update all call sites of `render_login_form` and `render_menu` to pass `theme`.
</action>
