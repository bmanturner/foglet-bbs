```elixir
defmodule Foglet.TUI.Screens.Domain do
  @moduledoc """
  Domain-module lookup helper for Foglet BBS TUI screens.

  One responsibility:

  1. **Lookup** — Given a `session_context` map and a domain key atom,
     returns the configured domain module or `{:error, :not_configured}`
     when the key is absent or the domain is not set up.

  Supported keys (locked in AUDIT-02):
    :boards, :threads, :posts, :markdown

  Callers provide `state.session_context` (the narrower input).
  Each call site is responsible for its own default-module fallback
  via an explicit `{:error, :not_configured}` branch. This keeps
  defaults visible and test-injectable at the call site.

  See `Foglet.TUI.Screens.BoardList`, `ThreadList`, `PostReader`,
  `PostComposer`, `NewThread` for call-site examples.
  """

  @supported_keys [:boards, :threads, :posts, :markdown]

  @type domain_key :: :boards | :threads | :posts | :markdown
  @type result :: {:ok, module()} | {:error, :not_configured}

  @doc """
  Returns `{:ok, module}` when `ctx` contains a domain module configured
  for `key`, or `{:error, :not_configured}` otherwise.

  `key` must be one of #{inspect(@supported_keys)}. Unknown keys always
  return `{:error, :not_configured}` — no raise.
  """
  @spec get(map(), domain_key()) :: result()
  def get(ctx, key) when key in @supported_keys do
    case get_in(ctx, [:domain, key]) do
      nil -> {:error, :not_configured}
      mod -> {:ok, mod}
    end
  end

  def get(_ctx, _key), do: {:error, :not_configured}
end
```

**Key implementation notes:**

1. **Two-clause `get/2`** — the first clause has a `when key in @supported_keys` guard so the
   compiler will warn if `@supported_keys` grows without the clause body being updated. The
   catch-all (`def get(_ctx, _key)`) returns `{:error, :not_configured}` for unknown keys — this
   honors D-02's "no raise on unknown key" rule.
2. **`get_in(ctx, [:domain, key])`** — returns `nil` both when `ctx` lacks `:domain` and when
   `ctx.domain` lacks `key`. Both paths map to `{:error, :not_configured}`. No separate case.
3. **No `Code.ensure_loaded/1`** — per D-02 and RESEARCH §7 landmine 13, configured modules are
   trusted. The `thread_list.ex` `function_exported?/3 + Code.ensure_loaded/1` pattern is a
   Phase-6 correctness fix, NOT a Phase 0 concern.
4. **No `defstruct`** — this is a stateless lookup helper; the only type needed is the return-value
   `result/0` type alias.
5. **No `alias` inside the module** — it is self-contained; no external modules referenced.
6. **Moduledoc style** mirrors `theme.ex`: one-sentence purpose, numbered responsibilities list
   (one item here), supported-key reference, footer referencing call-site files.
7. **Never nest multiple modules in one file** (CLAUDE.md gotcha — would cause cyclic dependency /
   compilation issues).

**Do NOT:**
- Add any second module in this file (no nested `defmodule`).
- Hide default modules inside `Domain.get/2` (D-06 — defaults stay at call sites).
- Return `nil` or raise for any input.
- Add `@dialyzer`, `credo:disable`, or `sobelow_skip` anywhere.
- Name the module file anything other than `domain.ex` — the mirror test path depends on it.
