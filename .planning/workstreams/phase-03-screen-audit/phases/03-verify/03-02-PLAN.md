| Boundary | Description |
|----------|-------------|
| terminal key event -> Verify screen handler | Untrusted user input can append/delete code chars or trigger submit/resend. |
| Verify screen handler -> Accounts domain | Submit/resend crosses into account verification and code-generation functions. |
| Login/Register entry -> Verify screen state | Screen transitions can accidentally preserve stale in-memory buffer/counters. |

| T-03-05 | Denial of Service | `resend_code_raw/1` | mitigate | Preserve `resend_cooldown_until` blocking and modal feedback; tests must cover blocked resend without mutation. |

| T-03-07 | Tampering | `submit_raw/1` | mitigate | Preserve `@max_attempts`/`cooldown_until` lockout after five invalid attempts; tests must assert attempts and future cooldown. |
