## Trust Boundaries

PostReader is a render/handler module invoked by `Foglet.TUI.App`. It reads
from `state` (already-authenticated session state), resolves domain modules
via `Foglet.TUI.Screens.Domain.get/2`, and returns new state + commands. No
new external surface is added by this plan. No user input crosses a new
boundary — the `@default_terminal_size` attribute is a static tuple literal
with no runtime input, and `init_screen_state/1` is called only from
trusted in-process callers (`get_screen_state/1` and, pattern-analogously,
`post_reader.ex` itself should future callers need it).

| Boundary | Description |
|----------|-------------|
| App → Screen | `Foglet.TUI.App.do_update/2` calls into `PostReader.handle_key/2`, `render/1`, `load_posts/2`, `flush_read_pointers/2`. Already a trusted in-process boundary pre-plan; unchanged by this plan. |
| Screen → Domain | `Domain.get(ctx, :posts|:boards|:threads|:markdown)` resolves a module atom from `session_context`. Already established pre-plan; unchanged. |
| Documentation artifacts | REQUIREMENTS.md and VERIFICATION.md are governance documents, not runtime artifacts. No security boundary. |
