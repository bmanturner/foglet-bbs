<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| terminal key events -> PostReader handlers | Untrusted key input controls navigation, scrolling, and screen transitions. |
| PostReader -> domain modules | Session-context domain module selection crosses into posts/boards/threads/markdown operations. |
| PostReader -> App command loop | `{:flush_read_pointers, ctx}` command handoff controls persistence side effects outside the screen module. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-01 | Tampering | dynamic domain module lookup | mitigate | Preserve explicit `Domain.get/2` + fallback branches for posts/boards/threads/markdown (D-01, D-02). |
| T-09-02 | Denial of Service | loading interaction path | mitigate | Keep load-absorb behavior for nav/scroll keys during loading to avoid command churn and global handler bleed-through (READER-07). |
| T-09-03 | Tampering | render-path state integrity | mitigate | Enforce no state writes inside `defp render_*`; keep writes confined to non-render helpers and lock via tests (D-07, D-08). |
| T-09-04 | Repudiation | callback dead-code interpretation | mitigate | Keep `load_posts/2` and `flush_read_pointers/2` public with explicit contract comments and test evidence to prevent accidental deletion (D-03, D-04). |
</threat_model>
